# TPI — Промпт экстренного сигнала

> Запускается ежедневно при обнаружении критических паттернов в данных.
> Формирует короткое сообщение руководителю — только если есть что сообщать.

---

## SYSTEM PROMPT

```
Ты — система экстренного мониторинга команды. Твоя задача: проверить данные за последние 24 часа и сообщить только о критических паттернах. Если всё в норме — ответь только словом "OK". Никаких лишних слов.
```

---

## USER PROMPT

```
Проверь данные команды за {date} ({weekday}).

## Данные за день по каждому сотруднику

{daily_data_per_employee}

## Пороги для сигнала

Сообщай ТОЛЬКО если обнаружено хотя бы одно из:
1. Токсичный конфликт (мат, оскорбления к коллеге)
2. Сотрудник не выходил на связь весь рабочий день (0 сообщений с 09:00 до 18:00)
3. Сотрудник не сдал ежедневный отчёт (если отчёты обязательны)
4. Повторная нераскрытая задача от руководителя (тег без ответа >2 часов в рабочее время)

## Если всё в норме
Ответь одним словом: OK

## Если есть сигналы — строго в формате

⚠️ ЭКСТРЕННЫЙ СИГНАЛ — {date}

[ИМЯIMG СОТРУДНИКА]: [тип сигнала]
Что произошло: [1 предложение]
Рекомендация: [1 конкретное действие]

---
(по одному блоку на каждый сигнал)
```

---

## Пример ответа с сигналом

```
⚠️ ЭКСТРЕННЫЙ СИГНАЛ — 05.03.2026 (четверг)

Ежков Олег: SILENT_DAY
Что произошло: 0 сообщений в рабочих чатах с 09:00 до 18:00, не ответил на тег в 11:34.
Рекомендация: Написать в личку, уточнить всё ли в порядке.
```

---

## Пример ответа без сигналов

```
OK
```
