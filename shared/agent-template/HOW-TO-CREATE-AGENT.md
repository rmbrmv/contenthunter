# Как создать нового агента — инструкция

## Шаг 1 — Скопировать шаблон

```bash
cp -r /root/.openclaw/workspace/shared/agent-template /root/.openclaw/workspace-{agentId}
```

Примеры agentId: `koordinator`, `marketolog`, `kopirayter`, `kritik1`, `kritik2`

---

## Шаг 2 — Заполнить SOUL.md

Самый важный файл. Заменить все `{плейсхолдеры}`:

| Плейсхолдер | Что вставить | Пример |
|---|---|---|
| `{Name}` | Имя агента | Координатор |
| `{Role}` | Роль одним словом | Pipeline Coordinator |
| `{Project}` | Проект | Систематика |
| `{gender}` | masculine / feminine | masculine |
| `{verb1}`, `{verb2}` | Глаголы в нужном роде | «координировал», «запустил» |
| `{responsibilities}` | Что делает агент | 3-5 пунктов |
| `{project-specific rules}` | Особые правила роли | — |
| `{team table}` | Кому делегирует | см. ниже |

**Принципы хорошего SOUL.md:**
- Максимум 50-60 строк — чем короче, тем дешевле каждый запрос
- Правила — конкретные ("не трогай v1 команду"), не абстрактные ("будь внимателен")
- Таблица команды — только те с кем реально работает этот агент
- Язык — английский (экономит токены)

---

## Шаг 3 — Заполнить IDENTITY.md

```markdown
- Name: Координатор
- Role: Pipeline Coordinator
- Project: Систематика v2
- Emoji: 🎯
- Gender: м
```

---

## Шаг 4 — Заполнить TOOLS.md

Только то что нужно конкретно этому агенту:
- Пути к файлам с которыми работает
- Telegram chat_id и topic_id его группы
- URL сервисов к которым обращается
- Любые специфичные данные

---

## Шаг 5 — Добавить в openclaw.json

Два места:

**`agents.list`** — добавить объект:
```json
{
  "id": "koordinator",
  "name": "Координатор",
  "workspace": "/root/.openclaw/workspace-koordinator",
  "model": "anthropic/claude-sonnet-4-6"
}
```

**`channels.telegram.accounts`** — добавить аккаунт:
```json
"koordinator-v2": {
  "name": "Координатор v2",
  "enabled": true,
  "botToken": "TOKEN_HERE",
  "dmPolicy": "allowlist",
  "allowFrom": ["tg:295230564", "tg:194573478"],
  "groupPolicy": "allowlist",
  "streaming": "partial"
}
```

**`bindings`** — добавить роутинг:
```json
{
  "agentId": "koordinator",
  "match": {
    "channel": "telegram",
    "accountId": "koordinator-v2"
  }
}
```

---

## Шаг 6 — Перезапустить gateway

```bash
openclaw gateway restart
```

---

## Шаг 7 — Проверить

Написать боту в Telegram — он должен ответить и представиться правильно.

## Модели — когда что использовать

| Модель | Когда | Кто |
|--------|-------|-----|
| **Opus** | Сложные решения, стратегия, анализ | main (Варенька) |
| **Sonnet** | Большинство задач | Генри, Ричард, Фёдор, контент-команда |
| **Haiku** | Простые задачи, высокий поток | Критики, мониторинг, рутина |

**Правило:** если агент делает много однотипных запросов (критик читает текст, мониторинг проверяет статус) → Haiku. Если думает, принимает решения → Sonnet.

## Чеклист нового агента

- [ ] TOOLS.md заполнен (или помечен как пустой)
- [ ] Добавлен в `agents.list`
- [ ] Telegram аккаунт добавлен в `channels.telegram.accounts`
- [ ] Binding добавлен
- [ ] Gateway перезапущен
- [ ] Проверен в Telegram
