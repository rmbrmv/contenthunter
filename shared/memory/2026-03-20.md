# Обновления продукта — 2026-03-20

## AutoWarm (delivery.contenthunter.ru)

### 1. Видеозапись экрана при фарминге
- При выполнении задачи фарминга автоматически записывается видео экрана телефона
- Запись загружается в S3 (save.gengo.io) и сохраняется в БД
- Отключение: env `FARM_SCREEN_RECORD=false`

### 2. ID задачи в заголовке лога
- Заголовок лога теперь: `📋 Лог — аккаунт (#ID)` вместо просто имени аккаунта
- Позволяет быстро идентифицировать задачу

### 3. Запись экрана как отдельное событие в логе
- Ссылка на запись экрана теперь отображается как кликабельная запись прямо в ленте событий (а не отдельным блоком вверху)
- Тип события: `screen_record`, meta содержит URL

### 4. Дедупликация выкладки — новая логика
- **Было:** проверка по `(content_id, pack_id)` — блокировала весь пак
- **Стало:** проверка по `(content_id, account_username, platform)` — одно видео нельзя публиковать дважды только на один и тот же аккаунт одной платформы

### 5. Фикс YouTube title/description
- При ручном добавлении в очередь сервер теперь автоматически подтягивает title, description, hashtags из validator_content
- YouTube: caption = title видео, описание = description контента + hashtags

### 6. Фикс URL-поллера для Instagram (критический баг)
- **Проблема:** задачи публикации Instagram зависали в `awaiting_url` — поллер не мог получить список видео с аккаунта
- **Причина:** функция `axios_fetch(url)` в server.js игнорировала второй параметр с заголовками. Instagram API требует заголовки `X-IG-App-ID` и специфический User-Agent — без них видео не возвращаются
- **Исправлено:** `axios_fetch(url, options)` теперь поддерживает кастомные headers и timeout
- Затронутые задачи (#244, #252) исправлены вручную
