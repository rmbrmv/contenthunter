## 2026-02-26: Бизнес-диагностика — глубокий системный анализ
**Что сработало:** Анализ 291 зума + 4755 сообщений + distribution DB → 15KB отчёт с корневыми причинами, количественными данными, графом зависимостей.
**Почему:** Роман ценит глубину и системность. Поверхностный первый драфт был отвергнут — переделал с данными.
**Паттерн:** Всегда подкреплять выводы цифрами из БД. Не «есть проблемы» → а «80 инцидентов оффлайна, 27 GEO-промахов, тел.78 в алертах 12 раз».

## 2026-02-27: Ферма-платформа — 7 экранов за 2 этапа
**Что сработало:** delivery.contenthunter.ru работает, Роман пользуется. Прогрев, Клиенты, Здоровье, Ёмкость, Целостность — всё на реальных данных из factory DB.
**Почему:** Чёткое ТЗ + делегирование Генри + итерации по фидбеку Романа.
**Паттерн:** Писать детальное ТЗ → делегировать Генри → показать Роману → итерировать. Не пытаться угадать с первого раза.

## 2026-02-27: Быстрая чистка данных по фидбеку
**Что сработало:** Когда Роман скинул скриншот с мусорными проектами/людьми — сразу почистил SQL, без длинных обсуждений.
**Почему:** Не спорил, не объяснял почему так вышло (это anti-pattern). Просто починил.
**Паттерн:** На критику данных — сначала фикс, потом объяснение. Роман ценит скорость реакции.

## 2026-02-28: Субботняя ревизия — формат с приоритетами
**Что сработало:** Недельный отчёт со светофором (🔴🟡🟢), топ-3 на следующую неделю, конкретные цифры по каждому пункту.
**Почему:** Структурированный формат позволяет Роману быстро сканировать и принимать решения.
**Паттерн:** Отчёты = светофор + цифры + конкретное предложение + «что берём?». Не стена текста.

## 2026-03-01: Месячная ревизия — системные риски + план
**Что сработало:** Февральский обзор: цифры месяца, 5 системных рисков, сделано/не сделано, P0/P1 приоритеты на март.
**Почему:** Формат «сверху вниз» — от цифр к рискам к плану. Роман видит полную картину.
**Паттерн:** Месячный отчёт = дашборд цифр → риски → прогресс → план. Каждый риск с квантификацией.

## 2026-03-03: Ревизия с конкретными номерами телефонов и процентами
**Что сработало:** Отчёт 3 марта — не абстрактно «проблемы с фермой», а: тел 119,140,145,158,163 офлайн в 04:00, заряд -1%, Instagram 150/229=65%, SipShine +2 лишних IG.
**Почему:** Конкретика позволяет действовать сразу — знаешь какой телефон чинить, какой аккаунт проверить.
**Паттерн:** В ревизии всегда указывать: номера телефонов, имена аккаунтов, ID клиентов, точные проценты. Абстракции бесполезны.

## 2026-03-05: Зумы включены в ревизию — сработало
**Что сработало:** После 3 дней прокрастинации обновил крон. Ревизия 5 марта уже содержит данные из зумов: «Роман+Данил: уникализация, модуль модерации», «Роман+Анна: распределение клиентов». Картина стала полнее.

## 2026-03-06: Рефлексия как триггер действий — цикл замкнулся
**Что сработало:** За 4-6 марта рефлексия перестала быть дневником. 4.03 — обновил крон. 5.03 — добавил whitelist. 6.03 — нечего фиксить (всё работает). Цикл «записал TODO → сделал в ту же рефлексию» стабилизировался.
**Почему:** Правило «если TODO второй раз — делай сейчас» сработало. Стыд от записи «опять не сделал» мотивирует.
**Паттерн:** Рефлексия должна заканчиваться действием, а не записью. Формат: прочитал → нашёл → СДЕЛАЛ → записал результат.

## 2026-03-11: Поиск встречи по искажённому голосовому вводу
**Что сработало:** Роман голосом попросил найти встречу: «выгрузке зубов славянск-гребацией». Расшифровал как «выгрузка зумов с транскрибацией». Нашёл за 3 SQL-запроса: schema → все за дату → фильтр по участникам (LIKE '%Анна%'). Определил что 2 записи (16+62 мин) = одна встреча по стыковке времён.
**Почему:** Не стал переспрашивать — интерпретировал + подтвердил. Поиск по участникам сузил выборку.
**Паттерн:** При голосовом вводе — не спрашивать «что ты имел в виду?», а интерпретировать + показать результат для подтверждения.

## 2026-03-11: Фокус саммари на запрошенной теме
**Что сработало:** Встреча 78 мин — куча тем (клиенты, ферма, баны, кадры). Роман попросил «часть про платформу». Выделил только это — структурировал как продуктовое саммари: проблема → решение → ценообразование → план перехода → команда.
**Почему:** Не вывалил транскрипцию, не пересказал всё подряд. Методолог фильтрует шум.
**Паттерн:** Саммари встречи = не пересказ, а извлечение решений по теме. Структура: проблема → решение → план → кто делает.

## 2026-03-12: Мультиисточниковый аудит = полная картина
**Что сработало:** Аудит Validator за одну сессию: SPEC.md + Puppeteer-скриншоты + API-данные + 16 клиентских чатов + factory DB. Каждый источник дал уникальный слой — документация = архитектура, скриншоты = UX, API = реальные данные, чаты = боли клиентов.
**Почему:** Роман ценит глубину. Одна точка зрения = поверхностно. 4-5 источников = системность.
**Паттерн:** Аудит продукта = минимум 4 источника: код/документация + UI + данные + голос клиента. Каждый отвечает на свой вопрос.

## 2026-03-15-23: Непрерывные ежедневные memory-файлы — 9 дней подряд

**Паттерн:** Ежедневный цикл: утром читаю sessions_history 2-3 ключевых агентов → пишу 5-10 строк в memory/ → отмечаю риски. Занимает 3 минуты, даёт непрерывный контекст.

**Что сработало:** Ревизия недели 7-14.03 отправлена в 09:00 MSK (по графику SOUL.md «суббота 12:00 MSK»). 10 пунктов: стратегические события + операционные проблемы + сравнение с прошлой неделей.

**Паттерн:** Клиентские коммуникации = я пишу полный драфт → Роман корректирует направление → я переписываю → FAQ проактивно. Не ждать идеальный бриф — итерировать.
