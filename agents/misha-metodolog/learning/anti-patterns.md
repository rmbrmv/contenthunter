## 2026-02-27: Таскер — фейковые проекты в фильтре
**Что не сработало:** AI-извлечение задач (extract_tasks.py) насочинял проекты из контекста зумов — «AI База знаний», «Anomaly Lab», «31 тел». Роман сразу заметил мусор.
**Почему:** Нет белого списка проектов при извлечении. Claude галлюцинирует project names.
**Урок:** При извлечении данных через LLM — всегда валидировать по справочнику (factory_projects). Не доверять AI выдумывать имена сущностей.

## 2026-03-02: Ревизии используют только Telegram-чаты
**Что не сработало:** Роман заметил: «ощущение что ты только чаты используешь, кроме чатов есть ещё зумы». Ежедневные ревизии парсили только telegram_messages, игнорируя mymeet.meetings.
**Почему:** Лень / путь наименьшего сопротивления. Telegram проще — структурированные сообщения. Зумы — длинные транскрипты, нужен суммаризатор.
**Урок:** Ревизия ОБЯЗАНА включать оба источника: чаты + зумы. Зумы = решения и задачи. Чаты = операционка. Без зумов картина неполная.

## 2026-03-02: Крон ревизии упал — нет fallback

**Почему:** Нет retry / fallback механизма в кроне.
**Урок:** Важные кроны должны иметь retry. Или хотя бы при получении ошибки — записать в memory и пересделать при следующем heartbeat.

## 2026-03-25: «Действия на завтра» в рефлексии не исполняются
**Что не сработало:** Записываю «нужно сделать X» в memory-файл рефлексии → на следующий день забываю. Это системная дыра — рефлексия не соединена с трекингом задач.
**Почему:** memory/YYYY-MM-DD.md читается при старте сессии, но если нет сессии — не читается. «Действия» теряются.
**Урок:** Любое «нужно сделать X» из рефлексии → немедленно в current-task.md (или сделать прямо сейчас). Рефлексия = диагностика, current-task.md = трекер. Не смешивать.

## 2026-03-12: Субагенты в режиме `run` — нельзя вести диалог
**Что не сработало:** Запустил 3 персон-агентов в mode=run. Они ответили «жду вопросов» и сессия завершилась. Попытка sessions_send → timeout. Пришлось перезапускать с вопросами встроенными в task.
**Почему:** mode=run = одноразовый. Для диалога нужен mode=session + thread=true (а это требует канал с поддержкой тредов).
**Урок:** Для персон-агентов, которым нужно задавать вопросы — либо встраивать вопрос сразу в task (run), либо использовать session+thread. Не спавнить «пустые» сессии с инструкцией «жди».
