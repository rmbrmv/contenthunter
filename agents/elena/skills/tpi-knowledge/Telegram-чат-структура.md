# Telegram-чат собственника — Структура топиков

> Группа с топиками (Topics). Добавляются боты агентов.
> Собственник взаимодействует с системой только через этот чат.
> Каждый топик — отдельный канал для своего типа информации.

---

## Название группы
**«[Название компании] · Елена HR»**

---

## Топики — 7 штук

---

### 📋 Дашборд
**Кто пишет:** ReportBuilder
**Когда:** каждый понедельник утром
**Что здесь:**
- HTML-файл с интерактивным дашбордом недели
- Текстовое резюме: TPI команды, кто вырос, кто просел, потери в деньгах
- Руководитель читает, не отвечает (топик только для получения)

**Правило:** один пост в неделю. Никаких обсуждений. Для вопросов — топик «Елена».

---

### ✅ Задачи
**Кто пишет:** ReportBuilder + Елена (при экстренных рекомендациях)
**Когда:** каждый понедельник + при экстренных сигналах
**Что здесь:**
- Список задач руководителю на неделю (максимум 5)
- Формат: → Имя: что сделать (причина)
- Руководитель отмечает выполненные ответом «✓» или «сделано»

**Пример содержимого:**
```
→ Сафаров: провести 1-on-1 сегодня (SILENT_DAY + TPI 4.5)
→ Геращенко: публично поблагодарить на созвоне (лучший рост недели)
→ Темирлан: поставить задачу с личной ответственностью (LOW_ENGAGE)
```

---

### 🚨 Сигналы
**Кто пишет:** AlertAgent
**Когда:** в момент срабатывания триггера (не ждёт понедельника)
**Что здесь:**
- Экстренные алерты: SILENT_DAY, ZERO_RESULT, TOXIC, SHARP_DROP
- Каждый алерт: имя → что случилось → цена вопроса → что делать сейчас
- Руководитель может ответить: «разобрался» или «@Елена — объясни подробнее»

**Правило:** алерты ночью не приходят (23:00–8:00), кроме конфликтов.

---

### 💬 Елена
**Кто пишет:** Елена (оркестратор) + руководитель
**Когда:** в любое время
**Что здесь:**
- Главный топик для диалога с системой
- Руководитель спрашивает что угодно: «что с Сафаровым?», «почему у команды TPI упал?»
- Елена отвечает по данным из системы
- Команды: `/статус`, `/команда`, `/[имя]`
- Елена здесь же сообщает о системных событиях: «данные за неделю собраны, отчёт готовится»

**Это основной канал общения — всё остальное автоматическое.**

---

### ⚙️ Настройка
**Кто пишет:** OnboardingAgent + руководитель
**Когда:** при первом запуске и при изменениях
**Что здесь:**
- Первый онбординг: интервью с OnboardingAgent
- Изменение конфига: новый сотрудник, изменение KPI, новые правила
- Команды: `/добавить сотрудника`, `/изменить правило`, `/пересмотреть KPI`
- После изменений — OnboardingAgent показывает обновлённый конфиг на подтверждение

**Правило:** изменения вступают в силу только после явного «да» или «подтверждаю» от руководителя.

---

### 📈 История
**Кто пишет:** MemoryKeeper (по запросу через Елену)
**Когда:** по запросу
**Что здесь:**
- Архив: все дашборды за предыдущие недели
- Графики тренда по конкретному сотруднику
- Список всех алертов с датами и статусами
- Руководитель спрашивает в топике «Елена» → Елена отправляет результат сюда

**Пример запроса:** «Покажи историю Сафарова за последние 8 недель» →
Елена запрашивает MemoryKeeper → результат появляется в «История»

---

### 🔧 Система
**Кто пишет:** Елена (технические уведомления)
**Когда:** автоматически
**Что здесь:**
- Технические статусы: «данные собраны», «таблица ФОТ не заполнена», «отчёт отправлен»
- Еженедельный пинг: «Пожалуйста, заполни таблицу ФОТ до пятницы»
- Напоминание об акцепте оценок (если руководитель не подтвердил)
- Ошибки интеграций: «CRM недоступна, жду ручного ввода»

**Правило:** это технический журнал, не для диалога. Если что-то требует внимания — Елена дублирует в «Сигналы».

## Сценарий первой недели

**День 0 (запуск):**
Собственник пишет в топик «Настройка» → OnboardingAgent проводит онбординг (~30 мин) → конфиг сохранён → система активирована.

**День 1–6:**
DataBridge собирает данные ежедневно в фоне.
Если срабатывает триггер → AlertAgent пишет в «Сигналы».
Системные уведомления идут в «Система».

**День 7 (понедельник следующей недели):**
Елена пишет в «Система»: «Данные за первую неделю собраны. Формирую отчёт.»
→ TPI рассчитан, оценки предложены
→ Елена пишет в «Елена»: «Первый отчёт готов. Проверь задачи на неделю.»
→ ReportBuilder отправляет дашборд в «Дашборд»
→ ReportBuilder отправляет задачи в «Задачи»
