# Предиктор и командный TPI

> Надстройки над ядром TPI. Не меняют формулу — добавляют аналитику тренда.

---

## Предиктор: флаги на основе тренда

Автоматические флаги выставляются по истории оценок:

| Флаг | Условие | Что делать |
|------|---------|-----------|
| ⚠️ RISK_OF_LEAVING | TPI↓ 3 недели + вовлечённость < 4 | Срочный 1-on-1, выяснить причину |
| 🌟 GROWTH_CANDIDATE | TPI↑ 3 недели + вовлечённость > 7 | Думать о повышении, дать ответственность |
| 📉 DECLINING | TPI↓ 2 недели | Наблюдать, подготовить 1-on-1 если продолжится |
| 📈 IMPROVING | TPI↑ 2 недели | Отметить прогресс |
| ➖ STABLE | Нет тренда | Штатный режим |

### Логика

```
IF tpi_week_3 > tpi_week_2 > tpi_week_1 AND engagement < 4:
    flag = RISK_OF_LEAVING

IF tpi_week_3 < tpi_week_2 < tpi_week_1 AND engagement > 7:
    flag = GROWTH_CANDIDATE
```

Для работы нужны минимум 3 недели данных. До этого — STABLE.

## Командный TPI

### Формула

```
TPI отдела = Σ(TPI сотрудников) / количество сотрудников в отделе
```

### Отделы (из team.employees)

| Отдел | Сотрудников |
|-------|-------------|
| ОФИС УК | 1 |
| Финансы | 2 |
| HR | 2 |
| Контент Hunter УК | 3 |
| Маркетинг / продажи | 2 |
| Клиентский сервис | 1 |
| Разработка | 2 |
| Обслуживание временных процессов | 6 |
| Филиалы ферм | 2 |
| YAVO | 1 |
| Relisme / Дизайн | 1 |
| Мими Зомби | 1 |
| Проекты Зуфара | 1 |

### В дашборде

TPI по отделам показывается в отдельной таблице:
- Сравнение отделов между собой
- Динамика vs прошлая неделя
- Если TPI отдела < 5 → сигнал руководителю направления

---

## Формат дашборда (обновлённый)

Все сотрудники показываются **всегда**, сгруппированные по категориям:

1. 🔴 **Требуют внимания** (TPI < 5 или падение > 1.5)
2. 🟡 **Наблюдение** (TPI 5–6.9)
3. 🟢 **Норма и выше** (TPI 7+)

Флаги предиктора ставятся рядом с TPI в таблице.
