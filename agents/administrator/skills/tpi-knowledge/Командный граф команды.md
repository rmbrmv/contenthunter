# Многомерный граф команды

> Инструмент визуализации командных паттернов на основе данных Telegram и Zoom.
> Не требует анкет и опросов сотрудников.

## Концепция

Один граф — шесть режимов просмотра. Каждый режим отвечает на отдельный управленческий вопрос.
Переключение между режимами меняет что означают рёбра и цвет узлов.

Источник вдохновения: фреймворк графов компании Фикус (Саша) — анкетирование команды с визуализацией ответов. Наша версия делает то же самое автоматически, без опросов.

## 6 режимов графа

### 💬 Коммуникация
**Вопрос:** Кто с кем чаще всего общается?
**Рёбра:** A ↔ B = частота @упоминаний в Telegram + совместные встречи в Zoom
**Применение:** Выявить изолятов, хабов, плотность командного взаимодействия

### 📋 Задачи  
**Вопрос:** Кто кому ставит задачи?
**Рёбра:** A → B = A поручил задачу B (направленные)
**Источник:** Telegram паттерны: «сделай», «нужно», «возьми», «подготовь», «проверь» + Zoom
**Применение:** Видеть реальных менеджеров vs исполнителей, перегруженных людей

### 🧠 Экспертиза
**Вопрос:** К кому реально идут за ответом?
**Рёбра:** A → B = A отправил кого-то к B как к эксперту (направленные)
**Источник:** «спроси у X», «X знает», «уточни у X» в Telegram + прямые вопросы в Zoom
**Применение:** Найти реальных экспертов (не по должности), знаниевые бутылочные горлышки

### 💥 Напряжённость
**Вопрос:** Где в команде есть трение?
**Рёбра:** A → B = A выражал негативный тон в адрес B (направленные)
**Источник:** Негативные слова рядом с @упоминанием: «ошибка», «почему не», «опять», «не сделал»
**Применение:** Выявить конфликтные пары, хронически проблемные взаимодействия

### 🔥 Выгорание
**Вопрос:** Кто показывает сигналы усталости?
**Визуализация:** Тепловая карта узлов (не рёбра) — цвет = уровень риска
**Источник без анкет:**
- Доля сообщений после 22:00 и до 08:00
- Доля сообщений в выходные дни
- Тренд активности: первые 4 недели vs последние 4 недели (падение > 40% = сигнал)
**Применение:** Превентивное выявление выгорающих до того как они уйдут

### ⛓️ Зависимости
**Вопрос:** Кто от кого зависит в работе?
**Рёбра:** A → B = A ждёт B чтобы продолжить (направленные)
**Источник:** «жду от», «блокируется на», «зависит от», «как только X сделает»
**Применение:** Найти узкие горлышки, критические зависимости, блокеров

## Ключевой инсайт

Пересечение режимов даёт диагностику скрытых паттернов:

| Паттерн | Комбинация режимов | Диагноз |
|---------|-------------------|---------|
| Высокая коммуникация + все задачи от одного | Коммуникация + Задачи | Узкое горлышко |
| Центр коммуникации, но периферия задач | Коммуникация + Задачи | Реальный лидер без полномочий |
| Высокая активность + сигналы выгорания | Коммуникация + Выгорание | Риск потери ключевого сотрудника |
| Изолирован в коммуникации, но источник задач | Коммуникация + Задачи | Тихий контролёр |

## Технические источники данных

| Слой | Telegram | Zoom |
|------|----------|------|
| Коммуникация | @упоминания | Совместные встречи |
| Задачи | Паттерны поручений | Распределение задач в речи |
| Экспертиза | «спроси у X», «X знает» | Прямые вопросы к персоне |
| Напряжённость | Негативные слова + @mention | Перебивания, несогласия |
| Выгорание | Временные паттерны | Падение % эфирного времени |
| Зависимости | «жду от», «блокируется» | Ожидание в речи |

## Ограничения

- **Личные договорённости** не видны (только публичные чаты)
- **Тихие сотрудники** слабо представлены в Telegram → компенсируется Zoom
- **Zoom транскрипты** содержат ошибки автораспознавания → достаточно для статистики
- **Задачи без трекера** — только явные текстовые поручения, неявные не попадают
