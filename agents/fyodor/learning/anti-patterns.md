## 2026-03-02: Скоринг лидов — ложные срабатывания
**Что не сработало:** Regex «продаж|оплат|клиент» поймал сообщения про оплату НАШИХ роликов (Ambassadori: «оплачено 6 роликов»)
**Почему:** Слова «оплата», «клиент», «продажа» многозначны — могут означать наш контракт
**Урок:** Всегда добавлять исключающий фильтр: `text !~* 'ролик|видео|контент'`. Перед записью скора — читать реальные сообщения руками

## 2026-03-02: План-скор для долгосрочных клиентов
**Что не сработало:** Сравнение всех накопленных просмотров из DB с планом текущего контракта из Sheets — для Shelkovitsa вышло 676%, для Flor 15% при одинаковой длине проекта
**Почему:** DB хранит ВСЕ просмотры за все контракты, план в Sheets — только текущего периода
**Урок:** Нужна дата начала текущего контракта. Без неё план-скор для клиентов 4+ недели некорректен. Уточнить у Романа источник дат контрактов

## 2026-03-02: Период отчёта без правила закрытого периода
**Что не сработало:** Изначально взял период «23 фев — 1 марта» вместо «21–27 фев»
**Почему:** Не знал правила: последний день = TODAY − 3 дня
**Урок:** ВСЕГДА первым делом считать period_end = TODAY − 3. Правило зафиксировано в SKILL.md

## 2026-03-02: Instagram 429 блокировка серверного IP
**Что не сработало:** Попытки логина в Instagram через Puppeteer с серверного IP — получали 429 Too Many Requests
**Почему:** Instagram агрессивно банит серверные IP без куков
**Урок:** Ждать 30+ минут (помогло). Для логина нужны cookies реального браузера. Публичные профили открываются без логина после снятия блока

## 2026-03-03: Неправильные JOIN в Distribution DB
**Что не сработало:** `JOIN factory_inst_accounts fia ON fia.project_id = fp.id` и `JOIN ... ON fia.id = pa.account_id` — колонки не существуют
**Почему:** Схема другая: `fia.pack_id → pa.id`, `fir.account_id = fia.instagram_id`, project_id живёт в `pack_accounts`
**Урок:** Правильная цепочка: `pack_accounts.project_id → factory_inst_accounts.pack_id → factory_inst_reels.account_id = fia.instagram_id → factory_inst_reels_stats.ig_media_id = fir.ig_media_id`

## 2026-03-03: Хардкод цифр в HTML-отчёте без проверки
**Что не сработало:** В дашборде было захардкожено "18 активных клиентов" — Роман сразу заметил несоответствие
**Почему:** При ручной генерации отчёта вписал цифру не перепроверив (реальность: 23 уникальных, 16 в скоринге, 25 в DB)
**Урок:** Все цифры в заголовочной статистике должны быть подтверждены конкретным SQL. Для "активных клиентов" — явно определить формулу и задокументировать в комментарии HTML

## 2026-03-11: Не прочитал скилл перед задачей — переделка
**Что не сработало:** Роман попросил "клиентский отчёт по форме для встречи с Кириллом" — я сделал индивидуальные текстовые карточки, хотя нужен был сводный HTML-дашборд по скиллу `client-weekly-report`
**Почему:** Не проверил доступные скиллы и существующие файлы (`ch_weekly_report.html`) перед выполнением
**Урок:** Для любого "отчёта/дашборда/формы" — СНАЧАЛА проверить `find workspace -name "*.skill" -o -name "*report*.html"`, прочитать скилл, потом выполнять. Переделка стоит доверия.

## 2026-03-11: Ограничение карточек без явного указания
**Что не сработало:** В задаче subagent написал "топ-12 клиентов" — Роман хотел все 22
**Почему:** Самостоятельно ограничил без согласования
**Урок:** Если пользователь сказал "все" — не ограничивать без явной причины. При генерации карточек клиентов всегда брать все активные проекты, если не оговорено иное.

## 2026-03-11: collected_at = MAX глобально обрезает старые ролики
**Что не сработало:** `WHERE s.collected_at = (SELECT MAX(collected_at) FROM factory_inst_reels_stats)` — показывало только ролики опубликованные в последние дни, старые ролики пропадали
**Почему:** Глобальный MAX собирает только дату последнего снимка; ролики старше этой даты выпадают
**Урок:** Для "последней статистики ролика" использовать `DISTINCT ON (ig_media_id) ORDER BY ig_media_id, collected_at DESC`, а не глобальный MAX

## 2026-03-03: Попытка искать в meeting_chunks для новых встреч
**Что не сработало:** Поиск по mymeet.meeting_chunks для сегодняшних встреч вернул 0 строк
**Почему:** Чанки создаются не сразу — новые транскрипты ещё не проиндексированы
**Урок:** Для встреч того же дня всегда читать content_text напрямую из mymeet.meetings, не через chunks/векторный поиск
