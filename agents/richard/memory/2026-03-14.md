## OpenClaw gateway onboarding — заметки

Из документации:
- После `npm install -g openclaw` нужно: `openclaw onboard --install-daemon`
- Это создаёт systemd сервис правильно
- Команда `openclaw gateway --port 18789` — неправильная, gateway запускается через systemd
- Правильно: `openclaw gateway install` → `systemctl --user enable --now openclaw-gateway.service`
- Или `openclaw gateway install` с sudo для system-wide
- Gateway требует интерактивный TTY для первого запуска (wizard)
