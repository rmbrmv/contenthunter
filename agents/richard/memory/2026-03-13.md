## ch-auth SSO — фиксы инвайт-флоу (12-13.03)

### Проблема 1: "Нет доступа" для незарегистрированных
- ch-auth блокировал всех кто не в users.json — а Алёна была новым пользователем
- Фикс: если есть `?redirect=` (кросс-авторизация через инвайт) → выдавать crossToken даже новому пользователю
- Регистрацию делает producer-copilot по инвайту, не ch-auth

### Проблема 2 (критическая): `_redirect` ломал verifyTelegramAuth
- Frontend передавал `{...tgData, _redirect: redirect}` в теле запроса
- `verifyTelegramAuth` считает HMAC по ВСЕМ полям кроме hash
- `_redirect` не входит в Telegram-подпись → checkStr не совпадает → hash ВСЕГДА invalid
- **Фикс**: `const { _redirect: redirect, ...tgData } = req.body` — деструктуризация ДО верификации
- Правило: любые кастомные поля нужно извлекать из req.body ДО передачи в verifyTelegramAuth
