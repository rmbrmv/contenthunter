## autowarm — фикс счётчика лайков фарминга (2026-03-25, от Генри)

**Проблема:** в таблице задач фарминга колонка ❤️ Лайки показывала 0, хотя лайки реально ставились в Фазе поиска.

**Root cause:** warmer.py писал `progress.likes = 0` из Фазы 2 (лента), перезаписывая лайки Фазы 0 (поиск).

**Фикс (коммит `ac129be`):**
- `progress.likes` теперь = сумма лайков из всех фаз
- В логе задачи теперь видно разбивку: `лайки=10/10 (лента=0, поиск=10)`
- UI (index.html): колонка `❤️ Лайки` читает `progress.likes` — данные уже корректные, без изменений фронта

**Для диагностики:** если лайки снова 0 — проверить, из какой фазы они ожидаются (лента/поиск/конкуренты). В `progress` есть поля: `likes` (итог), `search_likes` (Фаза 0), `competitor_follows` (Фаза 1).

## publisher.py — Instagram аудиодиалог + YouTube заголовок (2026-03-25, Даниил)

### Instagram: диалог «Название аудиодорожки» (коммит `3c0a09a`)
**Баг:** publish_tasks задача выходила без описания (задача #33).
**Цепочка:** caption введён → Поделиться нажато → появился ModalActivity «Название аудиодорожки» → код нажал `KEYCODE_BACK` → Instagram **отменил публикацию** → вернулся на caption screen с пустым полем → повторный Поделиться → видео без описания.

**Фикс:** убраны Назад/Cancel/KEYCODE_BACK из обработчика. Ищем Пропустить/Skip/Не сейчас.
**Страховка:** при возврате на caption screen — caption вводится заново автоматически.

**Правило:** в диалоге аудиодорожки Instagram — только Skip/Пропустить. BACK = отмена публикации.

## autowarm — универсальное сохранение суб-навигации (2026-03-25)

В delivery.contenthunter.ru добавлен механизм сохранения активного подраздела при F5.

**Функции в index.html:**
- `setSubParam(value)` — пишет `?sub=key:value` в хэш
- `getSubParam()` — читает `?sub=` из хэша
- `restoreSubState(sec, sub)` — восстанавливает таб. Принимает `sub` аргументом (не читает из URL) — иначе `nav()` затрёт его через `history.replaceState`

**Где вызывается `setSubParam`:**
- `upSwitchTab(tab)` → `up:queue` / `up:tasks`
- `_showSubBlocks(prefix, show)` → `wa:tasks`, `ig:accounts`, `tt:tasks` и т.д.
- `navAdmin(tab)` → `admin:users`, `admin:access` и т.д.

**Ключевое правило:** в `restoreNav()` всегда читать `savedSub = getSubParam()` **до** вызова `nav()`.

**Коммиты:** `b6495cf`, `c5d9cfa`, `851cd18` → GenGo2/delivery-contenthunter
