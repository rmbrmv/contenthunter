## Критический баг: параллельный запуск warmer-задач

**Проблема:** Запустил задачи #18 (Instagram), #19 (TikTok), #20 (YouTube) одновременно через INSERT. Задача #20 (YouTube warmer) автоматически поднялась фоновым процессом (`python3 -u warmer.py 20`) и перехватывала фокус экрана каждый раз когда Instagram пытался загрузиться.

**Симптом:** Instagram загружается, через 8 сек уходит в RecentsActivity → Launcher → YouTube. Warmer падает с "Приложение не загрузилось за 30 сек".

**Решение:** Перед запуском задачи проверять нет ли других warmer-процессов (`ps aux | grep warmer`). Запускать задачи строго последовательно, не параллельно.

**Фикс:** `kill <pid>` + `UPDATE autowarm_tasks SET status='cancelled' WHERE id=20` + `am force-stop com.google.android.youtube`.

**Также:** Таймаут `_wait_for_app` увеличен с 15 до 30 сек (патч в warmer.py).

**unic-worker:** pid 1523859 — `/root/.openclaw/workspace-genri/autowarm/unic-worker/worker.py` — это фоновый воркер autowarm, он подбирает pending задачи автоматически! Всегда отменять ненужные pending задачи перед запуском.

## ВАЖНЫЙ УРОК: pm list packages vs pm list packages -u
