## autowarm — фарминг: диагностика отсутствия лайков (2026-03-24, Даниил Павлов)

**Проблема:** все задачи фарминга с 23 марта показывали 0 лайков при нормальном количестве просмотренных видео.

**Причина (корень):** лайки в warmer.py ставились ТОЛЬКО если `detect_thematic()` возвращал True (видео признано релевантным по ключевым словам из БД). На новых аккаунтах лента не обучена → алгоритм показывает нерелевантный контент → совпадений с ключевыми словами нет → лайков 0.

**Дополнительная причина:** Claude fallback (`_analyze_with_ai`) возвращал 404 в старом процессе (проблема с ключом/версией). После рестарта PM2 работает корректно.

**Решение:** добавлена **Фаза 0 — Поиск по ключевым словам** (`run_search_phase`) в warmer.py.

### Фаза 0: Поиск (новая)
- Работает на первых 5 днях прогрева
- Берёт 1-2 случайных ключевых слова из `validator_brand_profiles` (БД)
- Открывает поиск на YouTube / TikTok / Instagram через ADB
- Вводит ключевое слово через ADBKeyBoard (поддержка кириллицы)
- Смотрит и лайкает 5-8 видео из результатов — гарантированно релевантные
- Возвращается в ленту рекомендаций

### Порядок фаз после изменения:
1. **Фаза 0** — Поиск по ключевым словам (обучение ленты)
2. **Фаза 1** — Конкуренты (подписки)
3. **Фаза 2** — Лента рекомендаций (основной цикл)

**Коммит:** `9d275ca` → GenGo2/delivery-contenthunter
**Файл:** `workspace-genri/autowarm/warmer.py`

### Правила/уроки:
- UI dump YouTube Shorts содержит мало текстового контента (только `content-desc` с описанием видео), и на новых аккаунтах там иностранный контент → текстовый матч не работает
- Без фазы поиска лента обучается медленно → лайков нет → прогрев неэффективен
- Фаза поиска должна идти ПЕРВОЙ, до конкурентов — иначе лента не получит сигнал о тематике
