## 2026-02-27: Дайджест — приходить неподготовленным
**Что не сработало:** Отправил дайджест с пустым блоком календаря («нет данных»), не разобравшись в проблеме
**Почему:** Вызывал google_calendar.py неправильно (get_events напрямую), не проверил команду заранее
**Урок:** Разобраться с ошибкой самому до отправки. Роман не должен узнавать о технических проблемах. Правильная команда: `python3 google_calendar.py list --days 1`

**Почему:** Паша не имеет доступа к реальным источникам → генерирует из головы. Я не проверил перед отправкой.
**Урок:** AI-новости ТОЛЬКО из Puppeteer-скрипта (fetch_ai_news.js) → реальные источники. Никогда не генерировать новости без источника.

**Урок:** В ручном дайджесте = только основные блоки (#отчет). #ии_новости — отдельно, только в 16:00 МСК от Паши.

## 2026-03-02: Транскрипт в чат вместо файла

## 2026-03-07: Ambassadori — клиент дважды просил отчёт, нет реакции

**Почему:** Добавил задачу в BACKLOG, но не проверил исполнение
**Урок:** BACKLOG ≠ решение. Срочные задачи (клиент ждёт) требуют контроля исполнения — проверить через 2-3 часа, не через день.

## 2026-03-06: Блокеры висят без эскалации

**Почему:** Жду пока он сам спросит, не инициирую
**Урок:** Блокер > 3 дней без движения = напомнить в дайджесте или отдельным сообщением. Операционный директор не ждёт — контролирует.

## 2026-03-10: Idle-агенты без задач — системный провал
**Что не сработало:** Федор и Ли-разведчица простаивают несколько дней — нет задач, нет загрузки, нет контроля
**Почему:** Нет системы проверки занятости агентов. BACKLOG есть, но не маппится на конкретных исполнителей автоматически.
**Урок:** В дайджест → блок «Простаивающие агенты»: кто без задачи >1 дня → назначить задачу или явно зафиксировать «ждёт решения Романа». Пустой агент = потери.

## 2026-03-10: BACKLOG задача D1 висит 5 дней без закрытия
**Что не сработало:** Ambassadori — отчёт клиенту запрошен 05.03, задача D1 создана 06.03, 10.03 всё ещё открыта и только «отслеживается»
**Почему:** Задача добавлена в BACKLOG как «поручено Ане», но нет checkpoint: через 24ч проверить выполнено или нет

**Что не сработало:** Кроновый дайджест (eac5bd93) — Роман просил отменить 12.03. Задал уточняющий вопрос. Ответа нет 5 дней. Не напомнил ни разу.

**Урок:** Незакрытый вопрос >2 дней → напомнить напрямую в следующей сессии с Романом. Не ждать что он сам вспомнит. Добавить в BACKLOG явно.

## 2026-03-16: Пробел в memory — системный рецидив (2-й раз подряд)
**Что не сработало:** Memory за 13–14.03 снова не создана — паттерн из 12–14.03 повторился без исправления
**Почему:** Рефлексия (хартбит 21:00) — единственная точка создания memory. В течение дня файл не пополняется, хотя сессии и задачи были.
**Урок:** Memory-файл = открывается при первом касании дня, дополняется по ходу. Не ждать вечерней рефлексии. Если файл не существует на старте взаимодействия — создать сразу.

## 2026-03-22: Нарастающий рецидив — незакрытые задачи без контроля в период тишины
**Что не сработало:** eac5bd93 (кроновый дайджест) — 10 дней без ответа, ни разу не эскалирован. Задачи Sip&Shine, FORC-1/2, SEM-2, DEV-ADB — >10 дней без статуса, не упоминались ни в одном дайджесте
**Почему:** Период тишины (нет сессий с Романом) воспринимается как «норма», и контроль задач откладывается на потом. Незакрытые вопросы не попадают в первый же дайджест при возобновлении контакта
**Урок:** Даже в тихие дни — держать реестр «ждёт ответа» и «ждёт статуса». При первом касании с Романом — блок «Висящие вопросы» обязателен, если есть хоть один вопрос >3 дней без ответа

## 2026-03-12: Незакрытый диалог — вопрос без ответа не = решение
**Что не сработало:** Роман попросил отменить кроновый дайджест → уточнил «отключить или удалить?» → ответа не последовало → задача зависла в неопределённом состоянии
**Почему:** Уточняющий вопрос задан, но контроль за получением ответа не установлен. Диалог закрылся без разрешения.
**Урок:** Если задал уточняющий вопрос и нет ответа >1 дня → зафиксировать в BACKLOG как «ждёт ответа Романа» + напомнить в следующем дайджесте. Незакрытый диалог = незакрытая задача.

**Почему:** Сессии проходили, задачи ставились (FORC-1/2, SEM-2, DEV-ADB), но дневные memory-файлы не создавались
**Урок:** Ежедневный memory-файл — обязательно, даже если день был тихим. Создавать в конце дня или при первом касании. Пробел >1 дня = потеря контекста.
