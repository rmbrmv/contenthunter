# Approved Patterns — Что сработало

> Записывай сюда каждый положительный фидбек от пользователя.

```
## YYYY-MM-DD: [Контекст задачи]
**Что сработало:** [Описание]
```

## 2026-02-27: Дайджест — формат без кода
**Что сработало:** Переход с кодовых блоков на обычный текст с **жирным** для чисел
**Почему:** Telegram рендерит bold в тексте, но не внутри ``` блоков
**Паттерн:** Дайджест = обычный текст + **жирные** цифры. Код только для SQL/скриптов.

## 2026-02-27: Средние по соцсетям вместо когорт
**Что сработало:** Убрал когорты (>500, 250-500...) → ср. просм. по платформам + % от позавчера + активность
**Почему:** Роман сам попросил и одобрил — нагляднее и операционно полезнее
**Паттерн:** Дистрибуция = ср. просм. по платформам + динамика + активность (лайки+комменты в одну колонку)

## 2026-03-02: RAG для поиска по встречам
**Что сработало:** mymeet.meeting_chunks + FTS/embedding поиск нашли встречу «Отчёт Внутренние проекты» с Никитой Пономарёвым
**Почему:** Роман помнил участника (Никита), а не название встречи — поиск по содержимому нашёл то что надо
**Паттерн:** Если встреча не найдена по title → искать по содержимому через meeting_chunks ILIKE или vector search

## 2026-03-02: Файл транскрипта вместо текста в чат
**Что сработало:** message(action=send, filePath=...) — Telegram принял файл корректно
**Почему:** Сырой транскрипт слишком длинный для чата, файл удобнее читать и сохранять
**Паттерн:** Транскрипты Zoom → всегда файлом (.txt), не текстом в чат

## 2026-03-03: Быстрый ответ «сколько зумов за день»
**Что сработало:** SELECT из mymeet.meetings WHERE DATE(created_at) = CURRENT_DATE → таблица с title/duration/word_count → краткий итог (N записей, N часов, группировка по участникам)
**Почему:** Один запрос даёт всю нужную картину, группировка по участникам добавляет контекст без лишнего
**Паттерн:** Вопрос «сколько зумов» → один SELECT + краткое саммари по участникам. Не перегружать деталями.

## 2026-03-06: Проактивный мониторинг чатов → задачи в BACKLOG
**Что сработало:** Без запроса Романа — сам проанализировал 217 сообщений из 20 чатов, извлёк 9 задач (D1–D9) по SMART, добавил в BACKLOG
**Почему:** Операционный директор не ждёт поручений — контролирует сам
**Паттерн:** Мониторинг чатов → если вижу проблему/задачу → сразу в BACKLOG + уведомление. Не ждать что Роман сам заметит.

## 2026-03-06: ICP-категоризация клиентов в Airtable
**Что сработало:** Заполнил 37 записей полем "ICP Категория" (7 переменных Да/Нет + 🟢🟡🔴) по когортной системе Кирилла
**Почему:** Данные из зума + чёткая система классификации → быстрое заполнение
**Паттерн:** Структурированные данные из встреч → сразу в Airtable/DB. Не хранить в голове/чате.

## 2026-03-07: Саммари YouTube-видео по запросу
**Что сработало:** Кирилл прислал ролик → сделал саммари с выводами конкретно для CH (не общий пересказ)
**Почему:** Адаптировал под контекст компании (OpenClaw, AI-агенты, корпоративная память)
**Паттерн:** Саммари = ключевые тезисы + «что это значит для нас». Не просто пересказ.

## 2026-03-10: Структура дайджеста — 6 блоков
**Что сработало:** Дайджест 10.03 покрыл все блоки: дистрибуция → клиентские инциденты → ферма → сервисы → активность агентов → открытые задачи + ближайшие события
**Почему:** Каждый блок даёт операционную картину своего домена — ничего не теряется
**Паттерн:** Структура утреннего дайджеста = 6 обязательных блоков. Пропустил блок → дайджест неполный. Проверять по чеклисту перед отправкой.

## 2026-03-10: Саммари внешнего контента = контекст CH
**Что сработало:** Саммари подкаста Кирилла (OpenClaw) сделано не как общий пересказ, а с конкретными применениями для CH (квалификация лидов, корпоративная память, обучение команды)
**Почему:** Контекст компании известен → адаптировать сразу, не ждать что Роман сам свяжет
**Паттерн:** Любой внешний контент (видео, статья) → саммари = «что это значит для CH» + конкретные применения

## 2026-03-04: Крон дайджест отработал полностью
**Что сработало:** Все 3 блока (#отчет, #бэклог, #развитие) отправлены автоматически без вмешательства
**Почему:** Шаблон чёткий, данные доступны, скрипты работают

## 2026-03-22: Рефлексия в период тишины — стабильность без сессий
**Что сработало:** Рефлексия 18–20.03 выполнена все 3 дня без пропуска — несмотря на полное отсутствие дневных сессий с Романом. Learning-файлы актуальны, дублей нет. 8 дней подряд непрерывной рефлексии.
**Почему:** Хартбит даёт независимую точку контроля — не зависит от активности пользователя
**Паттерн:** Рефлексия = хартбит-задача, не зависимая от дневных сессий. Тихий день ≠ пустая рефлексия. Зафиксировать: «сессий не было, рефлексия проведена» — достаточно.

## 2026-03-11: Рефлексия в рамках хартбита — проактивное обновление learning-файлов
**Что сработало:** Хартбит-задача «ежедневная рефлексия» запускает анализ memory/ и обновление learning-файлов без ручного запроса Романа
**Почему:** Системная рефлексия закрепляет паттерны до того, как они повторятся — не после
**Паттерн:** Хартбит-рефлексия = читай memory за 3 дня → что хорошо / что плохо / что повторяется → обнови оба learning-файла → зафиксируй в memory/сегодня.md. Без дублей.

## 2026-03-03: Уточняющий вопрос вместо угадывания
**Что сработало:** На «у меня 4 акка парсится» — спросил «где?» вместо того чтобы угадывать
**Почему:** Контекст был неоднозначным (ферма? amoCRM? Zoom?)
**Паттерн:** Если контекст неоднозначен — 1 короткий уточняющий вопрос. Не угадывать.

## 2026-03-02: Бриф по встрече — только из транскрипта
**Что сработало:** После замечания Романа («есть выдумка») — перечитал транскрипт строго, убрал всё что не звучало на встрече
