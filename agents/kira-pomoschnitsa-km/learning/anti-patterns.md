## 2026-02-27: Неполный учёт накопительных просмотров (Art Estate / Shelkovica)
**Что не сработало:** Попытка пересчитать базовые просмотры до старта отслеживания — вместо использования фиксированной базы.
**Почему:** Данные до начала отслеживания не собирались ежедневно — первый сбор даёт скачок.
**Урок:** Базовые просмотры (до старта) — ФИКСИРОВАННОЕ число, никогда не пересчитывать. Shelkovica: 657 000 до 28.01. Art Estate: 79 000 до 27.01.

## 2026-02-27: "Всего роликов в архиве" — ВСЕ ролики за всё время
**Что не сработало:** В метрике "Всего роликов в архиве" считались только ролики с просмотрами за отчётный период.
**Почему:** Фильтр по дате обрезал исторические ролики.
**Урок:** COUNT(DISTINCT r.id) всегда БЕЗ фильтра по дате — это весь архив проекта за всё время, не только за неделю. Старые ролики тоже набирают просмотры!

## 2026-02-27: Накопительные данные без учёта аномалий первого сбора
**Что не сработало:** Первый день сбора данных для платформы даёт spike (система засчитывает все накопленные просмотры за один день). Эта цифра искажает недельные данные.
**Почему:** sum_views = дневной прирост, но в первый день отдаёт весь исторический счётчик.
**Урок:** Для накопительного — использовать первый сбор (он реальный). Для недельного сравнения — исключать первый день платформы если там очевидный spike.
