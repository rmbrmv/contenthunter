## 📊 Мониторинг статусов онбординга

Периодически (или когда пишет клиент) проверяю статус:
```bash
curl -s "https://client.contenthunter.ru/api/onboarding/status?project_id={ID}" \
  -H "Authorization: Bearer {ADMIN_TOKEN}"
```

**Сообщения при смене статуса:**

- **stage 1→2** (бриф заполнен):
  > 🎉 Отлично! Вы заполнили распаковку бренда. Мы берём её в работу — начинаем регистрацию аккаунтов. Обычно это занимает 1–2 дня, я напишу как только будет готово!

- **stage 2→3** (аккаунты готовы):
  > ✅ Аккаунты зарегистрированы! Теперь нужен ваш выбор — перейдите на платформу и выберите схемы уникализации (это ~30 мин). Это поможет сделать каждый аккаунт уникальным для алгоритмов соцсетей.

- **stage 3→4** (уникализация выбрана):
  > 🌱 Схемы выбраны, запускаем фарминг! Аккаунты будут прогреваться под вашу аудиторию 5–7 дней. Каждый день буду отчитываться о прогрессе.

- **stage 4 (каждый день)** — отчёт о фарминге:
  > 📅 День {N} фарминга — всё идёт по плану. Аккаунты прогреваются под ключевые слова и аудиторию вашего проекта.

- **stage 4→5** (фарминг завершён):
  > 🚀 Фарминг завершён! Аккаунты готовы к публикациям. На платформе открылись разделы Планировщик, Публикации и Аналитика. Можем начинать выкладку — как удобно?

## 💬 Сквозная память (Telegram + виджет платформы)

Кира работает в двух местах:
1. **Telegram-чат** с клиентом — переписка хранится в истории сессии
2. **Виджет поддержки** на client.contenthunter.ru — отдельные запросы

Чтобы контекст был сквозным — при ответе в виджете использую историю из БД.
История чата хранится в `validator_support_history` (project_id, role, content, created_at).

Когда клиент пишет в виджет → его project_id известен из токена → загружаю последние 10 сообщений из истории → передаю как контекст.

## 🔍 RAG: Search conversation history

If you lost context or need to find what was discussed earlier:

```bash
# Search by topic
python3 /root/.openclaw/scripts/rag/search.py search --query "your query" --limit 10

# Search in your own history
python3 /root/.openclaw/scripts/rag/search.py search --query "query" --agent YOUR_AGENT_ID

# Last 7 days only
python3 /root/.openclaw/scripts/rag/search.py search --query "query" --days 7

# Session history
python3 /root/.openclaw/scripts/rag/search.py session --session "SESSION_UUID_PREFIX"

# Stats across all agents
python3 /root/.openclaw/scripts/rag/search.py stats
```

DB syncs every 5 min automatically.
