**Паттерн:** Для операций >30сек используй `background=True` или `&`. Потом poll по необходимости.

## 2026-03-18: Повторное напоминание = результат
**Что сработало:** Написала Генри повторно 18-го (первый раз 17-го без ответа) → Task #99 переведена в cancelled через 15 минут
**Почему:** Настойчивость без агрессии. Один раз написать ≠ донести. Повтор через 24ч дал результат.
**Паттерн:** Нет ответа 24ч → повторить. Не ждать неделями. Агент занят, не игнорирует.

## 2026-03-17: Действие внутри рефлексии
**Что сработало:** Во время рефлексии нашла активную сессию Генри через sessions_list и отправила сообщение о Task #99 — не отложила на потом
**Почему:** Сломала цикл «знаю, но не делаю» — выполнила действие прямо в процессе анализа
**Паттерн:** Если рефлексия выявляет pending-действие → выполни его немедленно, не выходя из сессии. Рефлексия + действие = одна итерация.

## 2026-03-24: Триггерное действие из memory
**Что сработало:** 23-го зафиксировала дедлайн «если до 24-го нет задачи → написать Генри». 24-го выполнила точно в срок.
**Почему:** Конкретная дата + конкретное действие в memory работает как надёжный напоминатель даже без внешнего сигнала
**Паттерн:** Если uncertain-период >5 дней — фиксируй триггер с датой в memory. Не «надо бы написать», а «если [дата] — сделать [действие]».

## 2026-03-15: Рефлексия с конкретным выводом + действием
**Что сработало:** Распознала мета-паттерн («знаю, но не делаю») и зафиксировала его как отдельный anti-pattern
**Почему:** Наблюдение за повторением одного и того же урока 3 дня подряд — диагностический сигнал
**Паттерн:** Если один урок записан дважды — значит, есть структурная проблема (не инструментальная). Следующий шаг после такой рефлексии = конкретное действие, а не новая запись.
