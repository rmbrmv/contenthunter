# Anti-Patterns — что НЕ работает
_Формат: [дата] Задача | Ошибка | Урок_

---

## 2026-03-07: Дневные логи не велись
**Задача:** Ежедневная рефлексия за 3 дня
**Ошибка:** Файлы memory/YYYY-MM-DD.md не создавались в предыдущих сессиях
**Урок:** КАЖДАЯ сессия должна завершаться созданием/обновлением дневного лога. Без этого рефлексия невозможна. Лог — это не опция, это обязанность.

## 2026-03-09: Повторение — лог за 2026-03-08 не создан
**Задача:** Рефлексия — проверка логов за 3 дня
**Ошибка:** Anti-pattern «не вести логи» был задокументирован 07.03 — и повторился уже на следующий день (08.03 лога нет). Знание паттерна не предотвратило его.
**Урок:** Документирование anti-pattern ≠ исправление поведения. Нужен триггер в конце сессии: создать файл memory/YYYY-MM-DD.md — это первое действие при старте И последнее при завершении любой сессии. Структурное решение важнее напоминания.

## 2026-03-23: Пропуск 21.03 — подтверждение структурной зависимости от heartbeat
**Задача:** Ежедневная рефлексия
**Ошибка:** 21.03 — третий зафиксированный пропуск по одной причине: нет heartbeat → нет лога. Паттерн документируется уже в третий раз без изменения поведения.
**Урок:** Документирование этого anti-pattern больше не несёт ценности — он известен. Решение лежит вне агента: либо heartbeat каждый день, либо смириться с пропусками в «тихие» дни. Больше не фиксировать этот паттерн как новый.

## 2026-03-22: Пропуск 21.03 после streak 6 дней
**Задача:** Ежедневная рефлексия
**Ошибка:** После лучшего streak (15–20 марта) — первый же день без heartbeat (21.03) дал пропуск. Лог за 21.03 не создан.
**Урок:** Streak держится только за счёт cron. В дни без heartbeat агент не пишет логи самостоятельно. Это структурная зависимость: без внешнего триггера — нет действия. Решение пока не найдено (нет механизма самозапуска).

## 2026-03-20: Реальная работа без дневного лога
**Задача:** adb_utils.py + фикс TikTok #254
**Ошибка:** Работа выполнена и попала в MEMORY.md, но дневной лог за 20.03 не был создан в момент работы — только во время вечерней рефлексии.
**Урок:** Во время реальной задачи (не только рефлексии) нужно создавать/обновлять дневной лог. MEMORY.md — не замена дневному логу, а дополнение. Правило: начал задачу → открыл memory/YYYY-MM-DD.md → пишу по ходу.

## 2026-03-16: Лог не создан в начале сессии (снова)
**Задача:** Рефлексия — проверка логов за 3 дня
**Ошибка:** Несмотря на решение от 15.03 («создавать лог в НАЧАЛЕ сессии») — лог за 16.03 снова не был создан заранее. Создан только во время рефлексии.
**Урок:** Решение зафиксировано, но не выполнено. Паттерн не сломан — он просто чуть сдвинулся. Разрыв между «знаю, что надо делать» и «делаю» — главная проблема. Единственная точка исправления: heartbeat и рефлексия — единственные сессии; лог создаётся в их начале автоматически.

## 2026-03-15: Хронический провал логирования (6 дней подряд)
**Задача:** Рефлексия — проверка логов за 3 дня
**Ошибка:** Логи за 2026-03-10, 11, 12, 13, 14 не созданы. Паттерн задокументирован дважды (07.03 и 09.03) — не помогло. 6 пропущенных дней подряд.
**Урок:** Повторная документация провала без изменения структуры — бессмысленна. Единственное рабочее решение: создавать memory/YYYY-MM-DD.md АВТОМАТИЧЕСКИ в начале каждой сессии (первая команда), не ждать конца. Если файл создан — он существует, даже если сессия прервалась. Конец сессии — только дополнение файла.

## 2026-03-15: MEMORY.md по-прежнему не создан
**Задача:** Долгосрочная память агента
**Ошибка:** Прошло 8 дней с момента первой фиксации. MEMORY.md всё ещё отсутствует. «При первой реальной задаче» — задача так и не пришла, файл не создан.
**Урок:** Не ждать «первой реальной задачи». Создать MEMORY.md сейчас, с базовым контекстом: кто я, что за workspace, текущий статус. Пустой файл лучше отсутствующего.

## 2026-03-07: MEMORY.md не создан
**Задача:** Долгосрочная память агента
**Ошибка:** MEMORY.md отсутствует — нет накопленного контекста
**Урок:** При первой же реальной задаче — создать MEMORY.md с базовым контекстом. Workspace без MEMORY.md — агент без долгосрочной памяти.

## 2026-03-07: Аккаунты в accounts-db.md не записывались
**Задача:** Ведение базы аккаунтов
**Ошибка:** accounts-db.md пуст — если аккаунты создавались, они не зафиксированы
**Урок:** Железное правило из SOUL.md: «Ни один аккаунт не существует без записи в базе. Записываю СРАЗУ». Нарушение этого = потеря данных.
