## 🎯 MyMeet RAG - Полная настройка

### Проблема
Роман хотел выгрузить все транскрипты из MyMeet (300+ встреч) в PostgreSQL с embeddings для RAG поиска.

### Решение

**1. Сбор ссылок на встречи:**
- Создан `mymeet_selenium_fixed.py` с автоматической пагинацией
- Скрипт кликает на кнопку "Следующая →" и собирает ВСЕ ссылки
- Результат: **213 уникальных встреч** (не 300, это всё что есть в MyMeet)

**2. Скачивание транскриптов:**
- Создан `mymeet_transcripts_v2.py` с улучшениями:
  - Клик на вкладку "Транскрипт" (не весь HTML)
  - Извлечение метаданных (название, дата, участники, длительность)
  - Сохранение в JSON (метаданные) + TXT (транскрипт)
  - Пропуск уже скачанных встреч (resume capability)
- Оптимизация: убрал сохранение HTML (~3x быстрее)

**3. PostgreSQL schema:**
- Создана схема `mymeet` (`sql/mymeet_schema.sql`)
- Таблицы:
  - `mymeet.meetings` - метаданные встреч
  - `mymeet.meeting_chunks` - chunks с embeddings (vector 384)
- Индексы: HNSW для векторного поиска + полнотекстовый search

**4. Загрузка в PostgreSQL:**
- Скрипт `mymeet_to_postgres.py` (аналог google_drive_sync.py)
- Embeddings: sentence-transformers (multilingual-MiniLM, та же модель что у Google Drive)
- Chunking: 1000 символов, overlap 100
- Команды:
  - `init` - создать таблицы
  - `sync` - загрузить транскрипты
  - `stats` - статистика
  - `search` - семантический поиск

**5. Документация:**
- Создан `/root/.openclaw/workspace/docs/MYMEET_RAG.md`

**Статус на конец дня:**
- ✅ Скрипты готовы
- ✅ Схема БД создана
- 🔄 Скачивание транскриптов в процессе (~213 встреч, ~2-3 часа)
- ⏳ Загрузка в PostgreSQL - после окончания скачивания

## 📝 Технические детали

### MyMeet Selenium fixes
- Проблема 1: Лимит 30 встреч → Добавлена пагинация (клик "Следующая")
- Проблема 2: Element click intercepted → JavaScript click + scroll to element
- Проблема 3: Popup блокирует кнопку → Добавлено закрытие popup перед началом
- Проблема 4: Не находит транскрипт → Добавлен клик на вкладку "Транскрипт"

### Оптимизации
- Убрано сохранение HTML (~3x быстрее, ~2x меньше места)
- Добавлена проверка существующих файлов (resume capability)
- Метаданные в отдельном JSON (удобно для PostgreSQL)

### Embeddings модель
Используется одна модель для всех RAG систем:
- `paraphrase-multilingual-MiniLM-L12-v2`
- 384 dimensions
- Русский язык ✅
- Бесплатная, работает локально

Это создаёт **единый векторный индекс** для:
- Google Drive документов
- MyMeet транскриптов
- Telegram чатов (когда настроим)

**Преимущество:** Можно искать по всем источникам одновременно!
