### ❌ Проблема с загрузкой MyMeet в PostgreSQL:

**Симптомы:**
- Python скрипты видят данные (291 встреча)
- `psql` запросы показывают 0 записей
- Данные **не persist** после закрытия соединения

**Попытки:**
- ✅ autocommit = True
- ✅ Явные COMMIT
- ✅ Разные пользователи (openclaw, postgres)
- ❌ Всё равно 0 записей в `psql`

**Гипотезы:**
1. Проблема изоляции транзакций (READ COMMITTED vs SERIALIZABLE)
2. Python подключается к другой базе/схеме
3. Конфликт между пользователями openclaw/postgres
4. Кеширование или VIEW не обновляется

**Следующий шаг:**
- Отладка PostgreSQL подключения
- Проверка search_path
- Возможно нужен явный VACUUM/ANALYZE

**UPDATE (вечер 22.02):**

```sql

  ├── people_aliases (152 alias, sources: mymeet, google_sheets)
  ├── people_ratings (0 строк, готова)
  └── meeting_participants (создана)
```

**2. Импорт команды из Google Sheets:**
- Таблица "Трэкшен веры" → 229 человек
- 52 человека с менеджерской иерархией (поле "Руководитель")
- Автоматический импорт alias-ов: "Кирилл" → "Попов Кирилл"

**3. Fuzzy matching:**
- **16 из 18** имён из MyMeet автоматически сопоставились (**89% успех**)
- Алгоритм: Levenshtein distance >85%
- Примеры: "Кирилл"/"Кирюша" → "Попов Кирилл", "Аня" → "Столярова Анна"

**4. Функции:**
- `resolve_person(name, context)` — умное сопоставление с учётом контекста
- Использует organizer встречи для disambiguation (несколько "Кирилл" → правильный по контексту)
