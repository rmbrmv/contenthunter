# 📊 Мониторинг топиков чата "Заказы"

Автоматическое создание новых клиентов в Airtable при появлении топиков в чате.

## Как работает

### 1️⃣ Отдел продаж создаёт топик в чате "Заказы"
- Название топика = название нового клиента
- Пример: "Content Hunter Россия"

### 2️⃣ Бот детектирует новый топик
- Проверяет, что это не системный топик
- Извлекает название клиента из топика

### 3️⃣ Автоматически создаётся запись в Airtable
- Таблица: "Брифы клиентов"
- Поле "Name": название клиента
- Статус: новая запись

### 4️⃣ Уведомление в production chat
```
🎉 Новый клиент от отдела продаж!

Клиент: Content Hunter Россия
Создан в Airtable автоматически из топика чата Заказы

Бот начал отслеживать этот топик для сбора дополнительной информации
```

### 5️⃣ Дальнейшее наполнение
- Бот продолжает слушать топик
- При появлении материалов/информации - дополняет запись
- Всё автоматически синхронизируется

## Настройка

### Шаг 1: Добавьте бота в чат "Заказы"

1. Откройте чат "Заказы"
2. Добавьте бота (@your_bot_name)
3. Дайте боту права администратора (для чтения сообщений)

### Шаг 2: Включите мониторинг топиков

В файле `.env` проверьте:

```bash
ENABLE_TOPIC_MONITORING=true
```

### Шаг 3: Перезапустите бота

```bash
./run.sh
```

## Проверка

### Тестирование

1. Создайте тестовый топик в чате "Заказы"
2. Назовите его "Тестовый клиент"
3. Проверьте:
   - Появилась ли запись в Airtable
   - Пришло ли уведомление в production chat

### Логи

Проверьте логи бота:

```bash
tail -f logs/bot.log | grep "топик"
```

Вы должны увидеть:
```
🆕 Обнаружен новый топик (клиент): Тестовый клиент
✅ Клиент 'Тестовый клиент' добавлен в Airtable
```

## Системные топики (пропускаются)

Бот автоматически игнорирует системные топики:

- "Общий"
- "General"
- "Администрация"
- "Тест" / "Test"
- "Вопросы" / "Help"
- "Важное"
- "Объявления"

Если в названии топика есть эти слова - запись НЕ создаётся.

## Как работает детекция названия

Telegram Bot API не предоставляет прямой доступ к названию топика по `message_thread_id`.

Поэтому бот использует несколько методов:

### Метод 1: Forum Topic Created (лучший)
Если сообщение содержит `forum_topic_created` - название извлекается оттуда.

### Метод 2: Первая строка первого сообщения
Если отдел продаж пишет название клиента в первом сообщении топика:

```
Content Hunter Россия
```

Бот берёт первую строку как название клиента.

### Рекомендация для отдела продаж

**Лучшая практика:**
1. Создайте топик с названием клиента
2. Первое сообщение - название клиента (если бот не подхватил автоматически)

## Дубликаты

Бот проверяет, существует ли уже клиент с таким названием в Airtable:

- ✅ Если клиент есть - запись НЕ создаётся
- ✅ Если клиента нет - создаётся новая запись

Логика:
```python
existing = airtable.search_by_name(client_name)
if existing:
    logger.info("Клиент уже существует")
    return
```

## Дополнительная функциональность

### Автоматическое наполнение

После создания клиента, бот продолжает слушать топик:

1. **Материалы** → поле "Материалы / заставки для соцсетей"
2. **Описания** → поле "Контент клиента"
3. **Ссылки** → автоматически детектируются

### Кэш обработанных топиков

Бот хранит в памяти список уже обработанных топиков:

```python
self.processed_topics = set()
```

Чтобы не создавать дубликаты при перезапуске - топики проверяются в Airtable.

## Отключение мониторинга

Если нужно временно отключить:

```bash
# В .env
ENABLE_TOPIC_MONITORING=false
```

Перезапустите бота:

```bash
./run.sh
```

## Troubleshooting

### Топик создан, но запись в Airtable не появилась

**Проблемы:**
1. Бот не запущен → запустите: `./run.sh`
2. Бот не в чате "Заказы" → добавьте бота
3. У бота нет прав → сделайте администратором
4. Название топика системное → проверьте список системных слов

**Проверка логов:**
```bash
tail -f logs/bot.log
```

### Создались дубликаты

**Причина:** Бот был перезапущен и кэш сбросился.

**Решение:** Проверка в Airtable предотвращает дубликаты. Если всё равно создались:
1. Удалите дубликаты вручную
2. Проверьте метод `search_by_name()` - работает ли корректно

### Неправильное название клиента

**Причина:** Бот взял первую строку сообщения, а там было что-то другое.

**Решение:**
1. Переименуйте клиента в Airtable вручную
2. Попросите отдел продаж писать название клиента в первой строке

### Уведомления не приходят в production chat

**Причина:** `PRODUCTION_CHAT_ID` не настроен.

**Решение:** Настройте согласно `NOTIFICATIONS_SETUP.md`
