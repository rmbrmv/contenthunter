# 🔔 Настройка уведомлений в production chat

Бот автоматически отправляет уведомления о новых материалах и обновлениях в производственный чат.

## Что бот уведомляет

### 1️⃣ Обновление брифов в Airtable
Когда бот детектирует материал и сохраняет его в Airtable, отправляется сообщение:

```
🔔 Обновление в брифах проектов

Обновлены данные: Content Hunter
Обновлённые пункты:
  • 📁 Материалы
  • 📝 Контент

🔗 Открыть в Airtable

Кому интересно — переходите смотреть! 👀
```

### 2️⃣ Детекция нового материала
Когда бот находит материал в клиентском чате:

```
📥 Новый материал от клиента

Клиент: Content Hunter
Тип: 🎬 video
Чат: Content Hunter - клиентский чат
Описание: Новый референс для видео

Материал сохранён в Airtable
```

## Настройка

### Шаг 1: Создайте production chat

1. Создайте новую группу в Telegram для команды производства
2. Добавьте в неё:
   - Клиентских менеджеров
   - Руководителя дистрибуции
   - Продюсеров
3. Дайте название (например: "Content Production Team")

### Шаг 2: Добавьте бота в production chat

1. Добавьте бота (@your_bot_name) в созданную группу
2. Дайте боту права администратора (нужно для отправки сообщений)

### Шаг 3: Получите Chat ID

**Вариант 1: Через команду /chatid**

1. Отправьте в production chat команду: `/chatid`
2. Бот ответит с ID чата
3. Скопируйте число (например: `-1001234567890`)

**Вариант 2: Через @userinfobot**

1. Переш решите сообщение из production chat боту @userinfobot
2. Он покажет Chat ID

### Шаг 4: Настройте .env

Откройте файл `.env` и добавьте полученный Chat ID:

```bash
# Notifications
PRODUCTION_CHAT_ID=-1001234567890
```

⚠️ **Важно:** 
- ID групповых чатов начинается с минуса (`-`)
- ID супергрупп начинается с `-100` (например: `-1001234567890`)

### Шаг 5: Перезапустите бота

```bash
./run.sh
```

## Проверка

После настройки:

1. Отправьте тестовый файл в любой клиентский чат (где работает бот)
2. Проверьте production chat - должно прийти уведомление
3. Если уведомление не пришло:
   - Проверьте, что Chat ID правильный
   - Проверьте, что бот администратор в production chat
   - Проверьте логи: `tail -f logs/bot.log`

## Отключение уведомлений

Чтобы временно отключить уведомления, закомментируйте или удалите `PRODUCTION_CHAT_ID` из `.env`:

```bash
# PRODUCTION_CHAT_ID=-1001234567890
```

## Кастомизация сообщений

Шаблоны уведомлений можно изменить в файле `notifications.py`:

- `send_airtable_update_notification()` - уведомление об обновлении
- `send_material_detected_notification()` - уведомление о материале

### Пример кастомизации

```python
# Изменить эмодзи для типов материалов

    'photo': '📸',
    'document': '📄',
    'link': '🔗',
    'text': '📝'
}
```

## Troubleshooting

### Уведомления не приходят

**Проблема:** Бот не отправляет уведомления

**Решения:**
1. Проверьте, что `PRODUCTION_CHAT_ID` установлен в `.env`
2. Проверьте, что бот администратор в production chat
3. Проверьте логи: `tail -f logs/bot.log`
4. Проверьте формат Chat ID (должен начинаться с `-`)

### Бот пишет "Forbidden: bot is not a member"

**Проблема:** Бот не может отправить сообщение

**Решение:** 
1. Убедитесь, что бот добавлен в production chat
2. Дайте боту права администратора

### Неправильный Chat ID

**Проблема:** Получили ID пользователя вместо ID группы

**Решение:**
1. Используйте @userinfobot
2. Перешлите ему сообщение **из группы**, а не от себя
3. Или используйте команду `/chatid` прямо в группе
