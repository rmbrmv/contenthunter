| Параметр | Google Vision API | OCR.space API |

## 🔄 Процесс обработки платежа (обновлённый)

```
1. Telegram → message_handler.py
              ↓
2. Скачивание изображения
              ↓
3. payment_ocr.py → OCR.space API (POST запрос)
              ↓
4. Парсинг JSON ответа → extracted text
              ↓
5. extract_payment_data() → базовые данные
              ↓
6. ai_parser.py → структуризация (Claude/GPT)
              ↓
7. sheets_writer.py → запись в Google Sheets
              ↓
8. Ответ пользователю
```

**Изменилась только часть 3:** Google Vision API → OCR.space REST API

## 💡 Рекомендации

### Для production использования:

1. **Мониторинг лимитов:**
   - OCR.space: 25,000 запросов/месяц (free tier)
   - При превышении можно купить платный план ($4.99/мес за 100k запросов)

2. **Оптимизация качества:**
   - Попросить клиентов делать чёткие скриншоты
   - Использовать оригинальные изображения (не пересланные в Telegram)
   - Для критичных документов можно добавить ручную проверку

3. **Fallback стратегия:**
   - Если OCR.space не распознал → базовое извлечение данных
   - Если базовое не сработало → AI парсинг со стандартным промптом
   - Если всё провалилось → попросить пользователя ввести вручную

4. **Логирование:**
   - Все OCR запросы логируются через `loguru`
   - При ошибках сохраняется traceback
   - Можно мониторить качество распознавания через логи
