## 14:00-14:17 UTC — Настройка Яны в группе «Инфлюенс»

### Проблема
Яна (@ch_influence_bot) не отвечала в новой группе «Инфлюенс». Роман добавил бота, тегал, делал админом — ничего.

### Диагностика
1. Privacy Mode — выключен ✅ (скриншот от Романа)
2. Бот в группе — да ✅
3. Gateway polling — получает updates ✅
4. **Причина: `reason: "not-allowed"` в telegram-auto-reply** — gateway дропает сообщения

### Корневая причина
- Нужен `groupAllowFrom` с ID отправителя для группового фильтра
- `groupPolicy: "open"` недостаточно без `groupAllowFrom`
- Группа должна быть явно прописана в `groups` конфига аккаунта

### Решение
- Нашла ID группы через debug-лог: **`-1003768225430`** (title: "Инфлюенс")
- Добавила в конфиг yan: `groups.-1003768225430.requireMention: false`
- Добавила `groupAllowFrom: ["tg:295230564"]`
- Debug logging: `logging.level: "debug"` + `logging.file` — мощный инструмент для диагностики

### Урок: Алгоритм добавления бота в новую группу
1. Добавить бота в группу в Telegram
2. Отключить Privacy Mode в BotFather (или сделать админом)
3. Узнать chat_id группы (debug log или getUpdates)
4. Добавить группу в конфиг: `channels.telegram.accounts.<bot>.groups.<chatId>.requireMention: false`
5. Добавить `groupAllowFrom: ["tg:<userId>"]` если не было
6. Перезапустить gateway
