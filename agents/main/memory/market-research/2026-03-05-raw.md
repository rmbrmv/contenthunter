### 2.1. 91eb87DdNgo — "My Openclaw REPLICA Runs a BOARD Meeting Now" (Goda Go, 1971 views, ER=8.42%)

**Ключевое содержание:**
- Автор построил систему "Autonomous Entrepreneur" — 7 специализированных агентов в Telegram группе
- Агенты: General (orchestrator/chief of staff), Research (react reasoning), Content (CMO), Finance (CFO с доступом к реальным revenue/transactions), Strategy (CEO — tree of thought reasoning), CTO (GitHub, PR, codebase), COO (operations, tasks, calendar), Critic (devil's advocate)
- Каждый агент живёт в отдельном Telegram-топике группы, каждый со своим ботом
- Агенты могут вызывать друг друга (invoke) — если Strategy нужен Finance, он напрямую его вызовет
- Критик — не может быть оптимистичным, использует Claude Sonnet 4.6 с дополнительным "brutally honest" промптом
- Board Meeting — собрание всех агентов для принятия бизнес-решений
- 300+ людей в сообществе уже строят свои версии
- Основная идея: вместо 10 отдельных AI-инструментов — единый "AI Second Brain" (Gobot, open source replica OpenClaw)
- Безопасность: всё на своём сервере, полный контроль инфраструктуры

### 2.2. 1EsQa1RH3RM — "OpenClaw Lobster Deep Dive" (OpenClaw_NF, 152 views, ER=1.32%)

**Ключевое содержание:**
- Lobster — нативный workflow engine OpenClaw (выпущен февраль 2026)
- Решает проблему непредсказуемости LLM-оркестрации в production
- Три проблемы стандартного подхода "LLM as orchestrator":
  1. Каждый шаг тратит токены на оркестрацию (даже если следующий шаг очевиден)
  2. Недетерминированность — один промпт может дать разную последовательность tool calls
  3. Накопление timeout errors — 20-шаговый pipeline с 2% ошибкой на шаге = 33% шанс провала
- Архитектура Lobster: "small CLI + JS pipes + approvals"
  - Каждый шаг = маленький CLI-скрипт (JSON in → one thing → JSON out)
  - Pipeline definition — структурированный файл с последовательностью шагов
  - Approval gates — паузы перед side effects (отправка email, GitHub comment, платёж)
  - Resume tokens — остановленный workflow можно продолжить позже
- Результат: детерминированное выполнение, меньше токенов, аудитируемый execution log
- Enterprise-ready: compliance team может подписать execution log
