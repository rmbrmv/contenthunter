## 📄 БЛОГИ / СТАТЬИ

### Substack: "The OpenClaw Settings Nobody Tells You About" (gobiraj)
**URL:** https://gobiraj.substack.com/p/the-openclaw-settings-nobody-tells

**Полный контент (важные инсайты):**

Автор запустил OpenClaw 24/7 на Raspberry Pi. Открыл биллинг и был шокирован расходами. Вот что нашёл:

**1. contextTokens лимит**
```json
"contextTokens": 80000
```
По умолчанию OpenClaw отправляет ВСЮ историю разговора на каждый запрос. Нужно ставить лимит. 80K — щедро для личного агента, можно ниже.

**2. Compaction mode: safeguard (не default)**
```json
"compaction": {
  "mode": "safeguard",
  "reserveTokensFloor": 24000,
  "memoryFlush": {
    "enabled": true,
    "softThresholdTokens": 6000
  }
}
```
- default = реактивный (ждёт пока история вырастет, потом одним куском)
- safeguard = проактивный (постоянно держит историю обрезанной через чанки)
- memoryFlush перед compaction записывает важный контекст в файл памяти

**3. Heartbeat model отдельно**
```json
"heartbeat": {
  "every": "1h",
  "model": "your-cheapest-capable-model",
  "target": "last"
}
```
Хартбиты по умолчанию используют primary model! 48 хартбитов/день × полный контекст = огромные расходы. Пин на дешёвую модель.

**4. Fallback chain — ловушка расходов**
```json
"model": {
  "primary": "your-primary-model",
  "fallbacks": ["your-fallback-model"]
}
```
Если primary бьёт rate limit, все запросы тихо переключаются на fallback. Надо знать стоимость fallback и мониторить.

**5. reserveTokensFloor**
```json
"reserveTokensFloor": 24000
```
Резервирует буфер для ответа модели. Без этого — context limit errors → retries → потраченные токены.
