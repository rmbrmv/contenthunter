- **Описание:** Два метода мультиагентных систем в OpenClaw: Route Method (через agents.md) и Terminal Method (openclaw agents add). Почему один агент ломается: context overload, prompt pollution, нет параллелизма, blast radius.
- **Транскрипт (полный):**
Most people using OpenClaw are doing it wrong. They have one agent trying to do everything, write code, check emails, manage files, review poll requests, and they're wondering why it keeps losing context and going off the rails. But what if instead of one overloaded agent, you had a team of specialists, each one laser focused on a single job, coordinated under one roof? That's exactly what a multi-agent system lets you do in OpenClaw.

Ключевые тезисы из транскрипта:
- Проблема 1: Limited context window — один агент тонет в информации, забывает инструкции, галлюцинирует
- Проблема 2: Prompt pollution — инструкции для разных задач конфликтуют ("be concise" vs "be thorough")
- Проблема 3: No parallelism — один агент делает одну задачу, несколько — параллельно
- Проблема 4: Blast radius — падение одного агента ломает всё vs изоляция
- **Method 1 — Route Method:** Создаёшь папки в проекте (check-email/, code-review/), пишешь .md с инструкциями для каждого суб-агента, добавляешь route-команды в agents.md. Подходит когда все агенты в одном проекте.
- **Method 2 — Terminal Method:** `openclaw agents add research-agent` — создаёт полностью изолированное рабочее пространство. Разные файлы, инструменты, контекст. Для enterprise-grade сетапов.
- Рекомендация: начинай с Route (2-3 агента), потом переходи на Terminal для изоляции. Продвинутые сетапы используют оба метода.

## 2. GitHub — официальные релизы

### v2026.3.2 (Latest) — 3 марта 2026
### v2026.3.2-beta.1 (Pre-release) — 3 марта 2026
### v2026.3.1 — 2 марта 2026

#### Ключевые изменения v2026.3.2:

**Новые фичи:**
- **Secrets/SecretRef полное покрытие:** 64 targets, runtime collectors, planning/apply/audit, onboarding UX. Unresolved refs fail fast.
- **PDF tool (first-class):** Нативная поддержка Anthropic и Google PDF, extraction fallback, настраиваемые defaults (pdfModel, pdfMaxBytesMb, pdfMaxPages).
- **Outbound adapters shared sendPayload:** Discord, Slack, WhatsApp, Zalo — multi-media iteration + chunk-aware text fallback.
- **MiniMax-M2.5-highspeed:** Новая модель в каталогах, онбординге, OAuth defaults.
- **Sessions/Attachments:** Inline file attachment для sessions_spawn (subagent), base64/utf8, lifecycle cleanup.
- **Telegram streaming по умолчанию:** `channels.telegram.streaming` теперь `partial` (было `off`).
- **Telegram DM streaming:** sendMessageDraft для приватного preview.
- **Telegram voice mention gating:** `disableAudioPreflight` для групп.
- **CLI config validate:** `openclaw config validate --json` — валидация конфигов до старта.
- **PDF diff output:** Качественная кастомизация для diff-артефактов.
- **Memory Ollama embeddings:** `memorySearch.provider = "ollama"` + fallback.
- **Zalo Personal plugin:** Полная перестройка на нативный zca-js.
- **Plugin SDK:** channelRuntime экспозиция, STT transcribeAudioFile, sessionKey в hooks, message lifecycle hooks.
- **Audio echo transcript:** Опциональная отправка транскрипта в чат до обработки.
- **Plugin runtime system/events:** requestHeartbeatNow, onAgentEvent, onSessionTranscriptUpdate.
- **CLI banner taglines:** Настраиваемые режимы (random/default/off).

**BREAKING CHANGES:**
- Onboarding defaults `tools.profile` to `messaging` (было broad coding/system)
- ACP dispatch теперь enabled по умолчанию
- Plugin SDK removed `api.registerHttpHandler` — нужен explicit `api.registerHttpRoute`
- Zalo plugin больше не зависит от внешних CLI

**Security (33+ fixes):**
- Gateway security hardening: loopback-origin, plugin HTTP auth, config backup permissions (0600)
- ACP sandbox inheritance: fail-closed для sessions_spawn с runtime="acp"
- Web tools SSRF guard: DNS pinning для web_fetch
- Browser security: fd-verified writes, output boundary hardening
- Webhook request hardening: auth-before-body parsing
- WS security: plaintext ws:// только loopback
- Nodes camera SSRF: bind to resolved node host
- И ещё ~20 фиксов
