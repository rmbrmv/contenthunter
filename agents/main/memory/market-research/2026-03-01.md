## 🔥 Фишки для внедрения

### 1. ACP Thread-bound агенты — первоклассный рантайм (v2026.2.26)
**Источник:** GitHub Releases  
**Суть:** В релизе v2026.2.26 ACP-агенты стали полноценными рантаймами для thread-сессий. Появились `acp spawn/send` dispatch integration, lifecycle controls, startup reconciliation, runtime cleanup, coalesced thread replies. До этого ACP был второсортным режимом.  
**Для нас:** Можно перевести всех 18 агентов на ACP thread-bound модель — получаем чище изоляцию сессий, контролируемый lifecycle, правильные replies в треды Telegram. Делегировать Генри, исследовать PR #23580.  
🔗 https://github.com/openclaw/openclaw/releases/tag/v2026.2.26

### 2. External Secrets Management — полный workflow (v2026.2.26)
**Источник:** GitHub Releases  
**Суть:** Введён полный secrets workflow: `openclaw secrets audit → configure → apply → reload`. Runtime snapshot activation — секреты подгружаются без рестарта. Strict target-path validation, ref-only auth-profile support, миграционный scrubbing.  
**Для нас:** Наши токены (Google Calendar, Telethon, PostgreSQL) сейчас лежат в файлах. Нужно мигрировать на новый secrets workflow — безопаснее, централизованно. Делегировать Плахову + Генри.  
🔗 https://github.com/openclaw/openclaw/releases/tag/v2026.2.26

### 3. Agents Routing CLI — управление биндингами (v2026.2.26)
**Источник:** GitHub Releases  
**Суть:** Появились команды `openclaw agents bindings`, `openclaw agents bind`, `openclaw agents unbind` для account-scoped route management. Channel-only → account-scoped upgrades, role-aware binding identity.  
**Для нас:** Можно настроить маршрутизацию запросов к агентам по ролям через CLI, а не руками в конфигах. Актуально при 18 агентах. Делегировать Плахову.  
🔗 https://github.com/openclaw/openclaw/releases/tag/v2026.2.26
