# Anti-Patterns — что НЕ работает
_Формат: [дата] Задача | Ошибка | Урок_

---

## 2026-03-03: WB orders — использование flag=1 для недельного среза
**Ошибка:** Первый запрос использовал `flag=1` с `dateFrom=2026-02-24` → вернул 9 записей вместо 86
**Почему не так:** `flag=1` фильтрует по `lastChangeDate`, не по дате заказа. Для недельного среза нужен `flag=0`.
**Урок:** `flag=1` — только для получения "что изменилось с момента X". Для аналитики периода — `flag=0`.

## 2026-03-03: Python — не проверял тип элементов JSON-ответа
**Ошибка:** `AttributeError: 'str' object has no attribute 'get'` — в списке были строки вместо dict
**Почему не так:** WB иногда добавляет строки-примечания в массив ответа.
**Урок:** Всегда фильтровать: `[s for s in data if isinstance(s, dict)]`

## 2026-03-03: Неверная интерпретация термина "продажи"
**Ошибка:** Показал выкупы (sales endpoint) когда пользователь имел в виду заказы (orders)
**Почему не так:** Не уточнил термин до начала работы — "продажи" в контексте WB-аналитика = заказы.
**Урок:** В начале сессии уточнять: "продажи = заказы или выкупы?" Для этого пользователя: продажи = заказы.

## 2026-03-03: Попытка несуществующих endpoints без проверки
**Ошибка:** Потратил 3+ итерации на `v2/nm-report`, `v3/nm-report`, `v1/search-report` — все 404
**Почему не так:** Пробовал угадать URL без проверки актуальной документации.
**Урок:** При неизвестном эндпоинте сначала проверить dev.wildberries.ru или попробовать известный рабочий эндпоинт из памяти.

## 2026-03-03: Игнорирование UTC→МСК конвертации
**Ошибка:** Фильтровал заказы по дате без учёта timezone. WB API отдаёт МСК, но граничные часы дня могут смещаться.
**Почему не так:** Не применил правило из wb-api-notes.md (UTC+3).
**Урок:** Для точной границы недели (пн 00:00 МСК) учитывать: в UTC это вс 21:00. Особенно критично для первого и последнего дня периода.
