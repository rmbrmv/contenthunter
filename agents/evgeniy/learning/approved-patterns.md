# Approved Patterns — что работает
_Формат: [дата] Задача | Результат | Почему сработало_

## 2026-03-03: WB Statistics API — получение заказов за период
**Что сработало:** `flag=0` с `dateFrom=2026-01-01` + фильтрация в Python по полю `date`
**Почему:** При `dateFrom` близком к нужной дате API возвращает 0 (баг/особенность WB). Широкий диапазон + Python-фильтр — надёжно.
**Паттерн:** Всегда брать данные с 01.01 текущего года, фильтровать самостоятельно.

## 2026-03-03: WB Sales Funnel API — правильная структура запроса
**Что сработало:** `selectedPeriod: {start: ..., end: ...}` (не `begin/end`, не `period`)
**Почему:** Документация WB неточная, нашли через итерацию по ошибкам от API.
**Паттерн:** При 400-ошибке читать `detail` — там WB указывает нужное поле.

## 2026-03-03: Дедупликация данных воронки
**Что сработало:** Дедупликация по `nmId` перед агрегацией (200 → 100 уникальных)
**Почему:** WB funnel API возвращает дубли при пагинации.
**Паттерн:** Всегда дедуплицировать по nmId после сбора всех страниц.

## 2026-03-03: Объяснение расхождения ЛК vs API
**Что сработало:** Чёткое объяснение причин (методология, таймзона, лаг) вместо "данные неверные"
**Почему:** Пользователь принял объяснение, доверие к данным API сохранилось.
**Паттерн:** Расхождение ЛК vs API ±10-20% — норма, объяснять, не паниковать.
