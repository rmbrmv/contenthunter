## 2026-03-23: Задачи unic_tasks зависают в processing (asyncpg JSONB → строка)

**Симптом:** задачи уникализации висят в `current_status='processing'` бесконечно.

**Причина:** asyncpg возвращает JSONB-колонку `meta` как строку → `dict(строка)` → ValueError → `mark_task_error` падает → задача не переходит в `error`.

**Диагностика:**
```bash
sshpass -p 'MNcwMPCiyiYtM5' ssh root@91.98.180.103 "pm2 logs unic-worker --lines 30 --nostream"
```

**Сброс зависших (экстренно):**
```sql
UPDATE unic_tasks SET current_status='pending', schemes_done=0, schemes_error=0, error_message=NULL
WHERE current_status='processing';
```

**Фикс в коде:** `3352057` — безопасный парсинг meta: строка → json.loads, dict → как есть, None → {}

**Урок:** если задачи в очереди не движутся — первым делом смотреть pm2 logs unic-worker, искать ValueError/Exception в обработчике ошибок.
