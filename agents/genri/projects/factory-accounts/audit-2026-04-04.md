## 1. Как это работает сейчас

### Архитектура

```
UI (index.html)
  └─ /api/factory/run (server.js)
       └─ spawn python3 account_factory.py --device ... --adb-port ... --task-id ...
            ├── create_gmail_browser()   — Chrome + OnlineSim SMS
            │    └── fallback: create_gmail()  — Settings → Add Account
            ├── register_instagram()
            ├── register_tiktok()
            └── create_youtube_channel()
```

### Файлы

| Файл | Роль |
|------|------|
| `autowarm/account_factory.py` | Основной пайплайн (1921 строк). Gmail + IG + TikTok + YT через ADB UIAutomator |
| `autowarm/gmail_factory_appium.py` | Альтернатива через Appium/ChromeDriver (не работает, задачи 7–13) |
| `autowarm/register_social.py` | Отдельный CLI-скрипт для регистрации соцсетей на готовый Gmail. **Не интегрирован в UI** |
| `autowarm/onlinesim.py` | API-обёртка OnlineSim (получение виртуального номера для верификации Gmail) |
| `autowarm/server.js` | Node.js API: `/api/factory/tasks`, `/api/factory/accounts`, `/api/factory/run`, `/api/factory/tasks/:id/log` |
| `autowarm/public/index.html` | UI-раздел `#factory` — вкладки «Аккаунты» и «Задачи», модал «Создать задачу» |

### Пайплайн Gmail (работает)

1. Генерируется случайный профиль (имя, фамилия, дата рождения, gmail-адрес, пароль)
2. Данные сохраняются в `factory_reg_accounts`
3. Открывается Chrome по URL `accounts.google.com/signup`
4. Поля заполняются по хардкоженным координатам SM-A175F (1080×2340)
5. Для SMS-верификации используется OnlineSim API (Россия/Казахстан, country=7/77)
6. Проверка успеха — `dumpsys account | grep type=com.google` + Vision fallback (Groq)
7. Делается скриншот-доказательство → `/tmp/gmail_proof/account_{id}.png`

### Пайплайн соцсетей (код есть, результата нет)

Функции `register_instagram()`, `register_tiktok()`, `create_youtube_channel()` написаны и вызываются в `run()`. Но **ни один аккаунт не прошёл дальше Gmail**.

## 3. Проблемы и незакрытые задачи

### 🔴 P0 — Критические (не работает)

#### 3.1 Instagram / TikTok / YouTube не регистрируются
**Симптом:** все 5 `done`-задач заканчиваются на `gmail_done`. Поля `ig_username`, `tiktok_username`, `yt_channel` в БД пустые у всех 13 аккаунтов.

**Вероятные причины:**
- `register_instagram()` падает на первом экране приложения (нет нужных кнопок в UIAutomator dump) и `print("WARNING: Instagram registration failed, continuing...")` проглатывает ошибку — задача помечается `done`
- Instagram в новых версиях требует телефонную верификацию через SMS — в `register_instagram()` используется только email-код из Gmail-приложения, но Gmail-приложение при первом запуске может требовать дополнительных шагов (setup wizard)
- TikTok DatePicker барабан не заполняется — в коде явно написан `# Просто тапаем Next без изменений (дата по умолчанию)` — там может быть стена до следующего экрана
- YouTube: `tap_content_desc(xml, 'Account')` — content-desc меняется между версиями YouTube

**Нужно:** добавить скриншоты в каждый шаг каждой соцсети для диагностики + тестовый запуск с `--step instagram`

---

#### 3.2 Зависшая задача #1 (running с 01.04)
**Симптом:** `id=1, status=running, current_step=gmail_name` — висит уже 3 дня.

**Проблема:** нет таймаута задачи и нет механизма автоочистки зависших задач. Задача всегда показывается как «В работе» в статистике раздела.

**Нужно:** `UPDATE factory_reg_tasks SET status='failed', error_message='timeout' WHERE id=1;` + добавить автотаймаут

---

#### 3.3 Задачи 7–13: Appium-ошибки
**Симптом:** 
- `ANDROID_HOME not set`
- `Could not find a connected Android device in 20000ms`
- `uiautomator2ServerInstallTimeout` / `uiautomator2ServerLaunchTimeout`

**Причина:** `gmail_factory_appium.py` (альтернативный через Appium) не работает в текущей среде. Appium не настроен: нет ANDROID_HOME, не установлен uiautomator2.

**Статус:** `account_factory.py` (основной, без Appium) работает стабильно для Gmail. Appium-скрипт не используется через UI, но задачи всё равно создавались.

### 🟡 P1 — Важные (мешают работе)

#### 3.4 Нет интеграции factory_reg_accounts → factory_pack_accounts
После создания аккаунтов они живут только в `factory_reg_accounts`. В `factory_pack_accounts` / `factory_inst_accounts` (которые используются фармингом и публикацией) они **никогда не попадают автоматически**.

**Нужно:** после успешной регистрации — либо автоматически создавать пак, либо кнопка «Добавить в пак» в UI

---

#### 3.5 Нет кнопки «Повторить» для упавших задач
Если задача упала на шаге `gmail_browser_dob` — нет способа перезапустить с этого шага через UI. Приходится делать вручную через CLI: `python3 account_factory.py --device ... --step instagram --task-id ...`

**Нужно:** кнопка «Retry» / «Повторить шаг» в строке задачи + выбор шага

---

#### 3.6 Нет проверки баланса OnlineSim перед запуском
`onlinesim.py::get_balance()` существует, но **нигде не вызывается** перед стартом задачи. При нулевом балансе — `get_number()` вернёт ошибку в середине пайплайна, Gmail не создастся, задача упадёт с `failed`.

**Нужно:** в `openCreateFactoryModal()` — показывать текущий баланс OnlineSim + блокировать кнопку если баланс < минимума

---

#### 3.7 Лог задачи виден только при создании
После закрытия модала «Создать задачу» — лог `_factoryLogInterval` останавливается. Чтобы посмотреть лог выполняющейся задачи — нет способа через UI.

**Нужно:** кнопка «📋 Лог» в строке задачи → выдвижная панель с поллингом `/api/factory/tasks/:id/log`

---

#### 3.8 Статистика считает задачи, а не аккаунты
Карточки (Всего / Выполнено / В работе / Ошибки) считают `tasks`, а не реально созданные аккаунты с соцсетями. Непонятно сколько «полных» аккаунтов (Gmail + все соцсети).

**Нужно:** добавить карточки по аккаунтам: Gmail, IG, TikTok, YT (счётчики с заполненными полями)

### 🟢 P2 — Улучшения

#### 3.9 register_social.py не интегрирован в UI
Скрипт `register_social.py` позволяет зарегистрировать соцсети на **готовый** Gmail (удобно когда Gmail уже создан вручную). Через UI это недоступно.

**Нужно:** в UI — возможность выбрать существующий аккаунт из `factory_reg_accounts` и запустить только шаги соцсетей

---

#### 3.10 Хардкоженные координаты под SM-A175F
Весь `account_factory.py` расписан под конкретную модель: `SM-A175F, 1080×2340`. Если используется другой телефон — координаты неверны.

**Нужно:** явно валидировать модель телефона перед запуском (`adb shell getprop ro.product.model`) и выдавать предупреждение если не SM-A175F

---

#### 3.11 OnlineSim API_KEY в коде
`onlinesim.py` строка 12: `API_KEY = "9efSzS9b2CtAB57..."` — хардкод в файле.

**Нужно:** перенести в `.env` (`ONLINESIM_API_KEY=...`)

---

#### 3.12 Нет поля «Статус» в таблице аккаунтов UI
В таблице аккаунтов показываются IG/TikTok/YT username + `socialStatus[x.ig_status]`, но нет общего статуса аккаунта и нет индикатора «полный» / «только Gmail».

---

#### 3.13 Нет удаления аккаунтов из UI
В `factory_reg_accounts` нет кнопки удалить/архивировать. Устаревшие аккаунты накапливаются.

## 4. Текущий workflow (как пользоваться тем что есть)

1. Зайти в раздел `#accounts/factory`
2. Нажать «+ Создать задачу» → выбрать телефон → нажать «🚀 Запустить»
3. Наблюдать лог прямо в модале (только пока модал открыт)
4. Через ~10 мин — Gmail создан, задача `done`
5. IG/TikTok/YT — **вручную через CLI** если нужны:
   ```bash
   cd /root/.openclaw/workspace-genri/autowarm
   python3 account_factory.py --device RF8YA0W57EP --adb-port 15068 --step instagram --task-id <N>
   python3 account_factory.py --device RF8YA0W57EP --adb-port 15068 --step tiktok --task-id <N>
   python3 account_factory.py --device RF8YA0W57EP --adb-port 15068 --step youtube --task-id <N>
   ```
6. После успеха — вручную добавить в пак для фарминга
