## 2026-04-01: pool_timeout + pool_recycle для FastAPI/asyncpg при нагрузке
**Что сработало:** `pool_timeout=10`, `pool_recycle=1800`, `max_overflow=10` в SQLAlchemy engine — validator перестал падать при нагрузке на PG. До этого: дефолтный pool_timeout=30с → uvicorn висел → PM2 крашил → 14 рестартов за день.
**Почему:** При нагрузке все соединения из пула заняты, новые запросы ждут 30с → timeout → краш. `pool_timeout=10` быстро отдаёт ошибку вместо зависания.
**Паттерн:** Для продакшн FastAPI с asyncpg: `pool_size=5, max_overflow=10, pool_timeout=10, pool_recycle=1800, pool_pre_ping=True`. Никогда не использовать дефолты в production.
