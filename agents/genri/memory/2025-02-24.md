## Virtual Office Dashboard — agent-office project

**Project**: Interactive web dashboard for Content Hunter's 8 AI agents as virtual office employees
**Location**: `/root/.openclaw/workspace-genri/agent-office/`
**URL**: `http://72.56.107.157:3847` (target: `office.contenthunter.ru`)
**Service**: `agent-office.service` (systemd, port 3847)
**Caddy**: reverse proxy on `office.contenthunter.ru` → localhost:3847

### Key decisions
- Canvas approach abandoned — blurry on mobile (devicePixelRatio issue), user said "качества нет"
- Switched to HTML/CSS for crisp rendering on all devices
- User rejected pure card-list dashboard ("зачем просто блоками, я хотел интерактив")
- Final approach: HTML/CSS positioned office scene with CSS animations
- Office scaled via CSS `transform: scale()` to fit any screen
- Added agent life simulation: walking to coffee/cooler/whiteboard/ping-pong, visiting each other, thought bubbles, typing animation

### Agent layout
- 4 desk islands (paired agents face-to-face): main+plahov, elena+fyodor, dasha+kira, oleg+genri
- Desks at y:300, h:130; agents positioned above (idx 0) and below (idx 1) desks
- Furniture: whiteboards, ping pong table, plants, coffee, water cooler, bookcase, clock

### Issues encountered
- Agents didn't render initially — z-index was too low (behind desks), positions were off
- Fixed: z-index:8 for agent-seat, repositioned desks to y:300 for more vertical space
- DNS for office.contenthunter.ru still propagating (A record set at Reg.ru)

### Current state (v8)
- Git: 8 commits on main branch
- Backend reads real session data from `/root/.openclaw/agents/{id}/sessions/*.jsonl`
- Status logic: <5min=working, 5-30min=waiting, 30-240min=idle, >240min=offline
