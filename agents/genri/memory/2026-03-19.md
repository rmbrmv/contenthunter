### Индивидуальные схемы уникализации по клиентам

**Задача от Даниила Павлова:** Уникализация должна идти только по одобренным клиентом схемам, а не по общим шаблонам.

**Что сделано:**

1. **ClientDashboard.vue (validator/frontend):**
   - При клике на пустой слот → проверка `GET /api/schemes/summary/{project_id}`
   - Если `approved < min_required` → модалка-блокировка: «У проекта "Название" не одобрено необходимое количество схем уникализации. Одобрите минимум N схем»
   - Кнопка «Перейти к одобрению схем» → `/client/schemes`
   - Работает сквозно для всех ролей (client, manager, producer, admin)

2. **autowarm/server.js — `runAutoUnicForDate()`:**
   - Заменён `SELECT FROM unic_schemes LIMIT N` → `SELECT FROM validator_scheme_preferences WHERE status='approved'`
   - Если одобренных < паков → пропуск с warning-логом
   - Если одобренных > паков → последовательная ротация (offset от последней задачи проекта, по кругу)
   - Если одобренных = паков → берём все

3. **autowarm/server.js — `GET /api/unic/schemes?project_id=X`:**
   - С project_id: возвращает только одобренные схемы (JOIN validator_scheme_preferences)
   - Без project_id: текущее поведение (все активные)

4. **autowarm/public/index.html — UI ручного создания задач:**
   - `unicOnProjectChange()` — при выборе проекта загружает одобренные схемы через `?project_id=X`
   - Предупреждение если 0 одобренных
   - `srcOpenUnicModal()` — то же для модалки уникализации исходников
   - `unicSubmitTask()` — теперь передаёт project_id
