### Массовый импорт стоп-слов в validator (2026-03-21)

**Задача от Даниила Павлова:** Добавить массовый импорт правил модерации (стоп-слов) из CSV или Google Sheets.

**Что сделано:**

1. **Backend (`admin.py`) — два новых эндпоинта:**
   - `POST /api/admin/moderation-rules/import/preview` — загрузка CSV/Google Sheets, парсинг, возврат заголовков и строк для маппинга. `full=true` возвращает все строки (для финального импорта), иначе первые 10 (превью).
   - `POST /api/admin/moderation-rules/import/run` — принимает `{rows, mapping}`, вставляет в БД, дедупликация по `word` (case-insensitive).

2. **Frontend — `ImportModerationModal.vue` (новый компонент):**
   - 3-шаговый флоу: источник → маппинг → результат
   - Поддержка CSV-файла и ссылки Google Sheets (public, export?format=csv)
   - Автомаппинг колонок по именам (word/слово, category/категория, severity, recommendation)
   - Превью таблицы, статистика результата (добавлено / дублей / ошибок)

3. **`ModerationRules.vue`** — кнопка `⬆ Импорт` в правом верхнем углу, открывает ImportModerationModal.
   - HELP_CONTENT обновлён — добавлена секция «Массовый импорт»

**Логика дедупликации:**
- Дубли по `word` (case-insensitive) пропускаются без ошибки
- `category` — пусто если не задано
- `severity` — дефолт `block`

### Фикс JSONB keywords в brand profile (2026-03-21)

**Проблема:** asyncpg не принимает Python list/dict напрямую в JSONB-поля — нужна явная сериализация через `json.dumps()`.

**Исправлено:** `brand.py` — `keywords` теперь сериализуется как `json.dumps(data.keywords) if data.keywords else None` (аналогично `social_links`, `extra`).
