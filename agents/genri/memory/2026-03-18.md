2. **Бэкенд (`routers/schedule.py`):**
   - Убрано ограничение «Cannot modify reach slots» — клиенты теперь могут загружать в reach-слоты

3. **Фронтенд (`ClientDashboard.vue`):**

### Онбординг новых клиентов (validator) — 03b2ce4

**Задача от Даниила + голосовое от Ромы:** Автоматизировать заведение клиентов. Раньше проект и юзер создавались отдельно вручную через админку. Рома хотел чтобы Кира (бот-КМ) или менеджер мог быстро завести клиента, даже с временным названием, и потом переименовать.

**Что сделано:**
1. **POST /api/clients/register** — атомарное создание проект + юзер:
   - Автогенерация `api_name` (транслит кириллицы → snake_case)
   - Автогенерация логина (`client_{api_name}`) и пароля (12 символов)
   - Проверка уникальности api_name и логина (суффиксы _2, _3...)
   - Если создаёт менеджер — проект добавляется в его `project_ids`
   - Доступ: admin + manager (ранее только admin)

2. **PUT /api/projects/{id}** — менеджер может редактировать свои проекты:
   - Только проекты из своих `project_ids`
   - `api_name` immutable для менеджера (защита связки с фабрикой)

3. **Фронт ManagerDashboard.vue:**
   - Кнопка «+ Добавить клиента»
   - Модалка: название проекта, имя, фамилия, email, telegram
   - После создания: карточка с логином/паролем + кнопка «Скопировать»

4. **Роут /clients** — доступен менеджерам (ранее только admin)

**Нюанс:** `validator_projects.id` не имеет autoincrement — пришлось делать `MAX(id)+1`

**Ключевое решение:** Для новых клиентов данных в Factory DB нет → `api_name` некритичен на старте. Свободное переименование `project` ничего не ломает (все связи по числовому `project_id`).
