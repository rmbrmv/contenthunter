## Snapshot 14:07–14:16 UTC — статус "processing" (обработка публикации)

### Уточнение от пользователя
Статус должен быть ПОЛНОСТЬЮ автоматическим:
- Видео отправлено → статус `processing` (обработка публикации)  
- Система сама ждёт пока соцсеть одобрит/опубликует
- После публикации — автоматически забирает ссылку на пост
- Подставляет в колонку "Публикация", статус → `done`
- Никакого ручного одобрения!

### DB
```sql
ALTER TABLE publish_tasks ADD CONSTRAINT publish_tasks_status_check 
  CHECK (status IN ('pending','running','done','failed','preflight_failed','paused','delegated','processing'));
-- moderation → processing (UPDATE 1)
```

### publisher.py — изменения

**`_save_post_url(url, final=False)`:**
- `final=False` → status='processing' (сохраняем профиль, ждём реальный URL)
- `final=True` → status='done' (финальный URL)

**`_update_post_url_final(url)`:** shortcut для final=True

**Новые методы:**
- `_read_clipboard()` — читает clipboard через `dumpsys clipboard`, извлекает URL соцсети
- `_share_and_copy_link(coords, text)` — тапает Поделиться → Копировать ссылку → clipboard
- `_auto_get_tiktok_url(wait_secs=45)` — ADB: профиль → видео → Поделиться → URL
- `_auto_get_youtube_url(wait_secs=90)` — ADB: канал Shorts → первый short → URL  
- `_auto_get_instagram_url(wait_secs=30)` — ADB: профиль → Reels → URL

**Порядок в publish_tiktok():**
1. Публикуем → _save_post_url(profile_url, final=False)  ← processing
2. _auto_get_tiktok_url(45) → если нашли → _update_post_url_final() ← done
3. Если не нашли — cron retry позже

**Аналогично для YouTube и Instagram.**

### server.js — cron checkProcessingTasks()

- Каждые **10 минут** проверяет задачи в status='processing'
- `scrapePostUrl(platform, account, profileUrl)` — веб-скрейпинг:
  - TikTok: `tiktok.com/@{user}` → парсит video ID из `"id":"XXXXXXX"` или href
  - YouTube: `youtube.com/@{handle}/shorts` → парсит `/shorts/VIDEO_ID`
  - Instagram: `instagram.com/{user}/reels/?__a=1` → парсит shortcode из JSON
- Если URL найден → UPDATE status='done', post_url=realUrl
- Первый запуск через 30с после старта сервера

**Подтверждено логами:**
```

```
