## HR платформа — требования от Романа (21:15 МСК, 26.02)

### 1. Производственный календарь в табеле
- Подгружать праздники/выходные по стране сотрудника (РФ или Казахстан)
- Сейчас февраль показывает 20 рабочих дней — должно быть 19 (по РФ)
- Праздничные дни = нерабочие, не учитываются в расчёте зарплаты
- API: можно использовать `data.gov.ru` (РФ) или `isdayoff.ru`

### 2. Автозаполнение табеля
- Система сама проставляет всем активным сотрудникам рабочие дни вперёд
- Дина и Роман вносят только **отклонения**: отпуск оплачиваемый, отпуск за свой счёт, больничный и т.д.
- Не заполнять каждый день вручную — только менять норму на отклонение

### 3. Физлица vs Сотрудники — РАЗНЫЕ сущности
- **Физлицо** — человек с персональными данными:
  - ФИО, город проживания, персональные данные
  - Одно физлицо может быть принято на **несколько должностей** в разные юрлица
  - Пример: Юта может работать в Релизме И в проектах Зуфара одновременно
- **Сотрудник** — запись о найме (должность + юрлицо + оклад + реквизиты)
  - К сотруднику привязываются **платёжные реквизиты** (не к физлицу)

### 4. Платёжные реквизиты на карточке сотрудника
Форма меняется в зависимости от типа оплаты:
- **Банковский счёт/карта** → поля: банк, номер счёта/карты
- **Наличные** → поля: кто передаёт, город
- **Криптокошелёк** → поле: адрес кошелька (или отметка "ситуативный" — каждый раз новый)
- Реквизиты подтягиваются при формировании заявки на оплату

### 5. Справочники — нужно расширить (детали позже)

### Итого — что переделывать в HR системе:
1. Производственный календарь (РФ + KZ) + интеграция в табель
2. Автозаполнение табеля рабочими днями + только отклонения вручную
3. Разделить физлица и сотрудников как отдельные сущности в БД и UI
4. Форма физлица: ФИО + город + персданные
5. Форма сотрудника: должность + юрлицо + оклад + динамические реквизиты по типу оплаты
6. Справочники — детали позже (Роман допишет)

### HR — дополнительные требования (очередь сообщений)

**6. Управленческая вкладка (для Кирилла и Романа)**
- Отдельная вкладка — оргструктура компании (по юрлицам/брендам)
- Автоматически отображает актуальные данные из системы (сотрудники, должности, ФОТ)
- История изменений: менялся оклад → записывается в лог
- Дополнительный столбец "план" — Кирилл может вносить будущие изменения (новая зарплата, увольнение, новая должность)
- Это черновик/планирование — не влияет на рабочие данные с Диной
- Дина/Роман могут видеть планы Кирилла и обсуждать

**7. Мультисессионный конфликт**
- Дина и Роман заходят по одному логину/паролю → конфликт сессий → данные пропадают, выкидывает одного
- Решение: создать отдельные логины (Дина / Роман / Кирилл) с разными ролями
- Или: сессии не конфликтуют (несколько активных сессий на один аккаунт)

**8. Удаление карточек сотрудников**
- Нет кнопки удаления — невозможно убрать дубликат или ошибочно созданного
- Пример: Дудникова и Дудников — дублирующиеся карточки, нельзя удалить
- Нужно: кнопка удаления в карточке сотрудника (с подтверждением)

**9. Форма бонусов / КПЭ / корректировок**
- Сейчас расчёт только оклад по табелю (с авансом или без)
- Нужна отдельная форма начислений/вычетов:
  - Выбираем сотрудника
  - Тип: бонус / КПЭ / надбавка / вычет / подотчёт
  - Сумма (+ или -)
  - Комментарий
  - Месяц
- Учитывается в расчёте зарплаты автоматически (плюс или минус к окладу)

**10. Мультивалютность**
- Оклады могут быть в RUB, KZT, USDT
- Управленческий учёт: всё конвертируется в RUB (для общего ФОТ в рублях)
- Выплаты: в реальной валюте сотрудника (или в USDT по текущему курсу)
- Курс: Google Finance или ЦБ РФ API + KZT/USD
- Нужна таблица курсов + автообновление

**11. Отдельная форма изменений по сотруднику (от Татьяны/Дины)**
- Не менять карточку сотрудника напрямую, а вносить изменения через отдельную форму
- Типы изменений: перевод на должность, смена отдела, смена компании/юрлица, изменение оклада (увеличение/сокращение/обнуление)
- Пример: у Бутовой с февраля оклад = 0, только бонусы — это фиксируется как изменение
- Все изменения пишутся в историю — можно посмотреть всю карьеру сотрудника
- Карточка сотрудника показывает актуальные данные, старые → в историю

**Татьяна (@Tattiyana)** — соавтор ТЗ по HR (Дина или отдельный человек — уточнить у Романа)

**12. Хранение документов (от Татьяны)**
- В карточке физлица: прикрепление файлов (скан паспорта, договор и т.д.)
- При изменениях сотрудника: прикрепить новый договор → автоматически в папку сотрудника
- Хранилище: Google Drive (папка на сотрудника) или S3
- Нужна форма загрузки файлов в карточке физлица и в форме изменений
