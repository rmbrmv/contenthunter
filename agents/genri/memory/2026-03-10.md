### Диагноз задачи #10 ("тест 1", .mov файл)
- FFmpeg работал нормально (validation OK)

- Причина: `s3.ru1.storage.beget.cloud` нестабильно рвёт SSL при загрузке результата
- Это httpx.RemoteProtocolError — сетевая флуктуация, не ошибка видео

### Исправления

**worker.py:**
1. `upload_to_s3()` теперь retries=5 с backoff (3×attempt сек) — FFmpeg НЕ перезапускается
2. `mark_task_error()` принимает `scheme_errors=dict` → сохраняет в `unic_tasks.meta.scheme_errors`
3. Частичная ошибка (не все схемы) → meta обновляется, задача всё равно `done`
4. `error_message` теперь содержит детальный текст по схемам: `"схема 3: S3 upload failed after 5 attempts: ..."`

**server.js:**
- `POST /api/unic/tasks/:id/retry` — сбрасывает статус на pending, очищает ошибки, убирает scheme_errors из meta
- Проверка: если уже processing → 409

**UI (index.html):**
- На задаче с ошибкой: отображается `error_message` + кнопка `🔄 Повторить`
- Кнопка "🔄 Повторить" в тексте ошибки И отдельно в блоке кнопок справа
- `unicRetryTask(id)` → POST `/api/unic/tasks/:id/retry`

**Задача #10:** сброшена на pending, воркер обработает повторно

### Модал «Добавить задачу выкладки»
Умный флоу:
1. **Проект** → загружает паки через `/api/publish/packs?project_id=X`
2. **Пак** → автозаполняет устройство (device_serial + adb_port из raspberry_port)
3. **Аккаунт** → автоопределяет платформу (Instagram/TikTok/YouTube)
4. **Платформа** → динамические варианты типа медиа (Instagram: reel/video/photo, TikTok: video, YouTube: short/video)

Поля формы: медиафайл (URL/path), caption, хэштеги, геотег, время публикации (datetime-local), протоколы прогрева до/после.

Восстановлены потерянные модалы: `modal-pub-events` (события задачи), `modal-approve-pub` (одобрение с ручным URL).

### Процесс публикации (publisher.py) — ключевые этапы
1. Preflight check (телефон онлайн, аккаунт совпадает, файл есть)
2. Разблокировка экрана через ADB
3. `adb push` медиафайла на устройство
4. Pre-warm (листание ленты N видео + пауза 30-90 сек)
5. Публикация через ADB (input tap, input text, am start)
6. Скриншот поста → `publish_tasks.screenshot_url`
7. Парсинг URL поста → `publish_tasks.post_url` (processing → done)
8. Post-warm (пауза 60-180 сек + ещё N видео)

### API эндпоинт `/api/publish/videos`
```sql
SELECT ur.id, ur.output_url, ur.created_at, ut.project_id, vp.project, uc.original_filename
FROM unic_results ur
JOIN unic_tasks ut ON ut.id = ur.task_id
LEFT JOIN validator_projects vp ON vp.id = ut.project_id
LEFT JOIN unic_content uc ON uc.id = ut.content_id
WHERE ur.status='done' AND ur.output_url IS NOT NULL
AND NOT EXISTS (SELECT 1 FROM publish_queue pq WHERE pq.unic_result_id=ur.id AND pq.status='done')
ORDER BY ur.created_at DESC
```
