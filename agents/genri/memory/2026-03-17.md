### Транскрибация аудио из видео (validator)

**Проблема:** Модерация проверяла только title + description. Если человек говорит запрещённые слова в видео, а в описании их нет — видео проходило.

**Что сделано:**
1. **60a4845** — Новый сервис `transcription_service.py`: извлекает аудио → Groq Whisper API (`whisper-large-v3`)
2. **95419de** — Фикс: отправляем MP4 напрямую в Groq (не WAV), фильтр мусорных транскрипций
3. **37bdf9f** — Карточка видео: показывает транскрипцию, запрещённые слова, рекомендации

**Pipeline валидации (финальный):**
1. Видео загружено → техпроверка (клиент + бэкенд)
2. Загрузка в S3
3. Скачивание с S3 → отправка MP4 в Groq Whisper → транскрипция
4. Модерация проверяет **title + description + transcript** на запрещённые слова
5. Результат: blocked/flagged/passed

**Конфиг:** `GROQ_API_KEY` в `.env` бэкенда. API: `https://api.groq.com/openai/v1/audio/transcriptions`, модель `whisper-large-v3`, язык `ru`.

**Ловушка:** Некоторые видео имеют битый AAC — ffmpeg не может декодировать. Groq при прямой отправке MP4 справляется лучше, но на битых файлах возвращает мусор «Продолжение следует...». Фильтруем такие ответы.

### Groq API key pool (round-robin)

**Задача:** Избежать упирания в rate limits Groq при массовой загрузке.

**Что сделано (dcf6a67):**
- `groq_keys.py` — менеджер пула ключей с round-robin
- При 429 → ключ на кулдаун 60 сек, запрос на следующий ключ
- 3 попытки (MAX_RETRIES) на запрос
- Конфиг: `GROQ_API_KEYS=key1,key2,key3` в `.env` (через запятую)
- Совместим с `GROQ_API_KEY` — все ключи в одном пуле

**Лимиты Groq (free tier, на ключ/день):**
| Whisper | 2K req, 28.8K audio sec |
| Vision  | 1K req, 30K tokens/min  |

**Инструкция для Даниила:** https://console.groq.com/keys → Create API Key → добавить в `.env` → перезапуск бэкенда

### Детекция водяных знаков (validator)

**Задача:** Блокировать видео с водяными знаками платформ (TikTok, CapCut, VK Clips и т.д.), но при этом не блокировать за наличие субтитров. Знаки могут перемещаться по углам кадра.

**Что сделано (ef147de):**
- Vision промпт обновлён: просит модель разделить текст на «СУБТИТРЫ» и «ВОДЯНЫЕ ЗНАКИ»
- `OcrResult` — структура с раздельными полями `subtitles` и `watermarks`
- Fallback-детекция: список известных паттернов (`tiktok`, `capcut`, `vk clip`, `likee`, `instagram`, `youtube`, `snapchat`, `reels`, `shorts`, `kwai`, `@username`)
- Водяной знак → **блокер** (status=blocked, score=0, rejected)
- Субтитры → идут только в модерацию на стоп-слова (не блокируются сами по себе)
- Дедупликация строк OCR (5bcde5d) — убирает повторы с соседних кадров
- UI: красный блок «🚫 Водяные знаки» с бейджами + рекомендация по удалению

**Итоговый pipeline валидации видео (полный):**
1. **Техпроверка** (клиент ДО загрузки + бэкенд ПОСЛЕ): размер, формат, кодек, ориентация, разрешение, аудио, тишина
2. **Загрузка в S3**
3. **Транскрибация речи** — Groq Whisper (`whisper-large-v3`)
4. **OCR кадров** — Groq Vision (`llama-4-scout`): 1fps → коллажи 3×2 → субтитры + водяные знаки
5. **Модерация текста** — title + description + transcript + субтитры → стоп-слова → blocked/flagged/passed
6. **Водяные знаки** — если обнаружены → blocked
7. **Виральность** — оценка потенциала
8. **Уникальность** — дедупликация

**Файлы:**
- `video_metadata.py` — техпроверка
- `transcription_service.py` — Groq Whisper
- `ocr_service.py` — Groq Vision + парсинг + детекция водяных знаков
