## Instagram caption fix (2026-03-24)

**Проблема:** видео публиковались без описания в Instagram.

**Root cause — 3 бага:**
1. XML-поиск EditText находил `TextView` (placeholder «Добавьте подпись»), тап по нему не фокусировал настоящий EditText
2. Instagram рендерит caption через WebView/Canvas — клавиатура не появлялась, текст уходил в пустоту
3. Верификация `caption[:20] in dump_ui()` — always false negative (WebView не отдаёт текст в XML) → лишний повтор ввода

**Фикс (коммит `432a6b8`):**
- Агрессивный поиск фокуса: 5+ координат кандидатов, тапаем пока не появится клавиатура
- Fallback: KEYCODE_TAB если все тапы провалились
- Убрана XML-верификация введённого текста (false negative → больше не делаем двойной ввод)

**Правило:** Instagram WebView не отдаёт введённый текст в dump_ui XML. Никогда не проверять наличие текста через dump_ui для Instagram caption.

## autowarm — фикс пагинации таблицы задач (2026-03-24, Юра)

**Проблема:** при переходе на 2-ю и последующие страницы таблицы задач фарминга система через ~10 сек возвращала пользователя на первую страницу.

**Причина:** `loadTasks()` вызывала `renderTasksTable(toRender)` без аргумента `page` → `_tasksPage = page ?? 0` → всегда сбрасывал на 0.

**Фикс (коммит `a3d4651`, delivery-contenthunter):**
- `loadTasks()` теперь вызывает `renderTasksTable(toRender, _tasksPage)` — передаёт текущую страницу
- Автообновление каждые 10 сек обновляет данные, но не сбрасывает страницу

**Правило:** при автообновлении таблиц с пагинацией — всегда передавать `_xyzPage` в `renderXyzTable(data, page)`. Сброс на 0 только при смене фильтров.

## autowarm — фарминг: фаза поиска по ключевым словам (2026-03-24, Даниил Павлов)

**Проблема:** все задачи фарминга с 23 марта ставили 0 лайков.

**Диагностика:**
- Лайки в `warmer.py` ставились ТОЛЬКО при `detect_thematic() = True`
- На новых аккаунтах лента показывает нерелевантный (иностранный) контент → ни одного матча по ключевым словам → 0 лайков
- Claude fallback (`_analyze_with_ai`) возвращал 404 (старый ключ в процессе). После рестарта PM2 исправлено.

**Решение:** добавлена Фаза 0 — `run_search_phase()` в warmer.py.

### Что делает Фаза 0:
- Работает первые 5 дней прогрева
- Берёт 1-2 случайных ключевых слова из `validator_brand_profiles`
- Открывает поиск (YouTube / TikTok / Instagram) через ADB
- Вводит слово через ADBKeyBoard
- Смотрит и лайкает 5-8 видео из результатов (гарантированно релевантные)
- Возвращается в ленту рекомендаций

### Новый порядок фаз:
0. Поиск по ключевым словам (обучение ленты) ← NEW
1. Конкуренты (подписки)
2. Лента рекомендаций (основной цикл)

**Коммит:** `9d275ca` → GenGo2/delivery-contenthunter
**Файл:** `autowarm/warmer.py`

**Правило:** без фазы поиска лента YouTube/TikTok/Instagram обучается медленно на новых аккаунтах. Фаза поиска обязательна в первые 5 дней.
