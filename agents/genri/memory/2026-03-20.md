### GitHub: автопуш через post-commit хуки

**Изменение от Даниила Павлова:** `git push` вручную больше не нужен.
- post-commit хук установлен во все 12 сервисов — после `git commit` автоматически пушит
- Если репо на GitHub не существует — хук создаёт приватный `GenGo2/<имя>`
- Ночной бэкап: `scripts/git_backup.sh` (03:00 UTC) — сканирует все `.git` папки
- При создании нового сервиса: `git init && commit` → хук сам всё сделает
- Переустановка хуков: `bash scripts/install_git_hooks.sh`
- Для кастомного имени репо → файл `.github-repo` в корне сервиса

**Правило:** убрал `&& git push origin main` из своего флоу, достаточно `git commit`.

### Фикс YouTube title/description при публикации

**Проблема (задача 251):** При ручном добавлении в очередь публикации YouTube видео получало description контента ("Тело должно быть в равновесии и гармонии") вместо title ("Симметрия") в качестве названия, а описание оставалось пустым.

**Причина:**
1. `POST /api/publish/queue/manual` не подтягивал title/description из validator_content — пробрасывал только то, что прислал UI
2. `publisher.py` при пустом description использовал fallback caption (=title) → дублировал заголовок в описание

**Исправлено:**
1. **server.js** — `POST /api/publish/queue/manual`: при наличии `unic_result_id` автоподтягивает `title`, `description`, `hashtags`, `geo` из `validator_content`. Формирует `caption`/`content_description` по платформе (YouTube: caption=title, content_description=description; Insta/TikTok: caption=description+hashtags)
2. **publisher.py** — убран fallback `caption→description` в обоих местах YouTube-флоу. Описание = только `description` + `hashtags`
3. Обновлены README.md, NOTES.md, справка в UI (HELP_CONTENT)

### Фикс axios_fetch — Instagram url-poller не находил видео

**Проблема:** задачи публикации Instagram (#244, #252) зависали в `awaiting_url`. URL-поллер не находил видео на аккаунте.

**Причина:** `axios_fetch(url)` в server.js принимал только URL, игнорируя второй аргумент с опциями. Instagram url-poller передавал `{ headers: igHeaders }` (с `X-IG-App-ID`, Instagram User-Agent), но заголовки молча отбрасывались. Без них Instagram API не возвращал видео.

**Исправлено:**
1. `axios_fetch` → `(url, options = {})` — кастомные headers мержатся с дефолтными, поддержан кастомный timeout
2. Задачи #244 и #252 вручную обновлены: URL подставлен, статус → `done`
3. Документация обновлена: README.md (новая секция URL Poller), NOTES.md

### Архитектурная доработка: day-in-day уникализация + дата публикации в Исходниках

**Задача от Даниила Павлова:** Если клиент загрузил видео в тайм-слоты на сегодня и они были одобрены сегодня — уникализация должна запускаться немедленно, а не ждать крон.

**Что сделано:**

1. **autowarm/server.js — GET /api/unic/sources:**
   - Добавлен `LEFT JOIN LATERAL validator_schedule_slots` → поле `publish_date` (дата ближайшего слота)

2. **autowarm/public/index.html — UI Исходники:**
   - Новая колонка «Дата публ.» после «Загружено»
   - Сортировка по publish_date
   - Если дата = сегодня → оранжевый цвет + bold (подсветка)
   - colspan обновлён 9→10 во всех местах

3. **autowarm/server.js — POST /api/unic/trigger-immediate:**
   - Новый эндпоинт, принимает `{ content_id }`
   - Определяет "сегодня" по timezone из unic_settings
   - Ищет слоты на сегодня для этого контента (filled + approved + passed, без дубликатов)
   - Если найдены → вызывает `runAutoUnicForDate(today, settings)`
   - Fire-and-forget: ошибки не блокируют одобрение

4. **validator/backend/src/services/delivery_webhook.py:**
   - `notify_content_approved(content_id)` → POST в autowarm trigger-immediate
   - httpx async, timeout 10s, ошибки логируются но не бросают

5. **validator/backend/src/routers/content.py:**
   - Хук после ручного одобрения менеджером → `notify_content_approved()`

6. **validator/backend/src/routers/validation.py:**
   - Хук после автоматического одобрения (полная валидация) → `notify_content_approved()`

**Архитектура:**
```
Клиент загружает видео в слот на сегодня (validator)
  → Менеджер одобряет или автовалидация проходит
  → notify_content_approved(content_id)
  → POST /api/unic/trigger-immediate (autowarm)
  → runAutoUnicForDate(today) → создаёт unic_task
  → unic-worker обрабатывает
  → assignUnicResultsToQueue → publish_queue → публикация
```

### Критическое правило
> **НИКОГДА не делать CREATE OR REPLACE VIEW без проверки что объект с таким именем не является TABLE.**
> Всегда проверять: `SELECT relname, relkind FROM pg_class WHERE relname='...'`
> Всегда делать pg_dump ПЕРЕД DDL-изменениями.
