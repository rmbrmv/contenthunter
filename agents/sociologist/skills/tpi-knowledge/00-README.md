## Архитектура в двух словах

```
Источники данных
  [чаты] [встречи] [отчёты] [CRM/таскер] [опросы]
         ↓
    DATA ENGINE
  (нормализация, расчёт метрик, граф коммуникаций)
         ↓
    JSON-метрики
  (TPI, sentiment, activity, reliability, expertise)
         ↓
    Агенты (параллельная интерпретация)
  [HR-директор] [Администратор] [Социолог] [Аналитик] [Коуч]
         ↓
    Рекомендации руководителю
```

## Ключевые принципы v2

1. **Data Engine — единственный источник правды.** Агенты не собирают данные, они интерпретируют метрики.
2. **Event-driven обновления.** Система реагирует на события, а не опрашивает источники по расписанию.
3. **Confidence scores.** Каждая метрика имеет уверенность (0–1), агент учитывает её при интерпретации.
4. **Feedback loop в Data Engine.** Коррекции руководителя улучшают алгоритм, а не отдельных агентов.
5. **Immutable data store.** Сырые данные не изменяются — только добавляются новые записи.

## Требования для развёртывания

- LLM API (OpenAI GPT-4o / Anthropic Claude 3.5+)
- Python 3.10+ для Data Engine
- PostgreSQL или SQLite для хранения метрик
- Telegram Bot API для чата
- Доступ к источникам данных (Bitrix24, CRM, таскер)
