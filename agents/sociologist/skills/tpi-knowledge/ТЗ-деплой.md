## 2. Технологический стек

### Язык и рантайм
- **Python 3.11+**
- Причина: лучшая экосистема для LLM-интеграций, NLP, data-обработки

### Telegram-интеграция
- **python-telegram-bot 20+** (async)
- Один бот-токен, маршрутизация по `message.message_thread_id` (topic_id)
- Topics требуют группу с включёнными темами (Forum group)

### LLM-вызовы
- **Anthropic SDK** (Claude Sonnet — основной)
- Обёртка: `llm_client.py` с единым интерфейсом → легко сменить провайдера
- Retry с экспоненциальной задержкой (3 попытки)
- Таймаут на вызов: 30 секунд

### Планировщик
- **APScheduler 3.x** (AsyncIOScheduler)
- Задачи:
  - `weekly_cycle` — каждый понедельник 9:00 (timezone из company_config)
  - `daily_collect` — каждый день 00:30
  - `fot_reminder` — каждую пятницу 17:00
  - `alert_check` — каждые 30 минут в рабочие часы (8:00–22:00)

### Хранилище данных
- **SQLite** (Starter) → **PostgreSQL** (Pro+)
- Схема: 5 таблиц (см. раздел 5)
- Бэкап: ежедневный дамп в папку `/backups/`

### Внешние интеграции
- **Google Sheets API** (gspread) — таблица ФОТ и план-факт
- **amoCRM API** — опционально в Starter, обязательно в Pro
- **Telegram Bot API** — чаты команды (бот добавляется в рабочие чаты)

### Инфраструктура
- **VPS**: 2 CPU, 2 GB RAM, 20 GB SSD — достаточно для 1–5 клиентов
- **ОС**: Ubuntu 22.04 LTS
- **Процесс**: systemd service или Docker container
- **Логи**: loguru → `/logs/elena_{date}.log`, ротация 7 дней

## 6. Telegram: настройка группы

### Шаги для клиента
1. Создать Telegram-группу
2. Включить: Настройки → Topics → Включить темы
3. Создать 7 топиков с названиями (порядок важен — id присваивается по порядку):
   - 📋 Дашборд
   - ✅ Задачи
   - 🚨 Сигналы
   - 💬 Елена
   - ⚙️ Настройка
   - 📈 История
   - 🔧 Система
4. Добавить бота в группу как администратора
5. Права бота: читать сообщения, отправлять сообщения, отправлять файлы
6. Передать разработчику: chat_id группы + topic_id каждого топика

### Получение topic_id
Переслать любое сообщение из топика боту @userinfobot → он вернёт message_thread_id

### Маршрутизация в коде
```python
TOPICS = {
    "dashboard":  int(os.getenv("TOPIC_DASHBOARD")),
    "tasks":      int(os.getenv("TOPIC_TASKS")),
    "alerts":     int(os.getenv("TOPIC_ALERTS")),
    "elena":      int(os.getenv("TOPIC_ELENA")),
    "setup":      int(os.getenv("TOPIC_SETUP")),
    "history":    int(os.getenv("TOPIC_HISTORY")),
    "system":     int(os.getenv("TOPIC_SYSTEM")),
}

async def route_message(update: Update):
    topic_id = update.message.message_thread_id
    if topic_id == TOPICS["elena"]:
        await orchestrator.handle_dialog(update)
    elif topic_id == TOPICS["setup"]:
        await onboarding_agent.handle(update)
    # ...
```

## 7. Чтение чатов команды

### Проблема
Telegram Bot API не позволяет читать историю чатов напрямую — только получать новые сообщения через webhook/polling.

### Решение для Starter
Бот добавляется в рабочие чаты команды как **обычный участник** (не администратор).
Все новые сообщения сохраняются в БД через webhook.
При сборе данных за неделю — DataBridge читает из БД, не из Telegram API.

```python
# Сохранение каждого сообщения из рабочих чатов
async def save_team_message(update: Update):
    if update.effective_chat.id in TEAM_CHAT_IDS:
        await db.save_message({
            "chat_id":     update.effective_chat.id,
            "user_id":     update.effective_user.id,
            "username":    update.effective_user.username,
            "text":        update.message.text,
            "date":        update.message.date,
            "message_id":  update.message.message_id
        })
```

### Ограничение
Система видит только сообщения с момента добавления бота. История до этого момента — недоступна.
Решение: при онбординге предупреждать клиента, что первая полноценная неделя — следующая.

## 8. Интеграция с Google Sheets (таблица ФОТ)

### Структура таблицы (даётся клиенту как шаблон)
```
| Неделя        | Сотрудник          | Оклад (₽) | Премия (₽) | Итого (₽) | Комментарий |
|---------------|-------------------|-----------|------------|-----------|-------------|
| 09.03-15.03   | Попов Кирилл      | 120000    | 0          | 120000    |             |
| 09.03-15.03   | Абрамов Роман     | 95000     | 15000      | 110000    | Премия за проект |
```

### Код чтения
```python
import gspread
from google.oauth2.service_account import Credentials

def read_fot_table(sheet_url: str, week_start: str) -> list[dict]:
    creds = Credentials.from_service_account_info(SERVICE_ACCOUNT_JSON)
    gc = gspread.authorize(creds)
    sheet = gc.open_by_url(sheet_url).sheet1
    rows = sheet.get_all_records()
    return [r for r in rows if r["Неделя"] == week_start]
```

## 9. Обработка ошибок и отказоустойчивость

### Правило: система не падает из-за одного агента
```python
async def run_agent_safe(agent_fn, *args, fallback=None):
    try:
        return await asyncio.wait_for(agent_fn(*args), timeout=60)
    except asyncio.TimeoutError:
        logger.error(f"{agent_fn.__name__} timeout")
        return fallback
    except Exception as e:
        logger.error(f"{agent_fn.__name__} error: {e}")
        await notify_admin(f"Agent error: {agent_fn.__name__}: {e}")
        return fallback
```

### Таймауты по агентам
| Агент | Таймаут |
|---|---|
| DataBridge | 10 мин |
| TPICalculator (1 сотрудник) | 60 сек |
| AlertAgent | 30 сек |
| ReportBuilder | 2 мин |
| OnboardingAgent | без таймаута (диалог) |

### Retry для LLM
```python
@retry(stop=stop_after_attempt(3), wait=wait_exponential(min=2, max=10))
async def llm_call(messages: list) -> str:
    response = await anthropic_client.messages.create(...)
    return response.content[0].text
```

### Что логируется
- Каждый запуск агента: старт, конец, длительность
- Каждый LLM-вызов: токены, стоимость, модель
- Каждый алерт: триггер, время, отправлен/нет
- Каждая ошибка интеграции: источник, текст ошибки

## 10. Безопасность

### Что защищаем
- Данные сотрудников — только внутри БД, не в логах
- API-ключи — только в .env, не в коде
- Доступ к боту — только из разрешённых chat_id

### Проверка отправителя
```python
ALLOWED_CHAT_IDS = set(map(int, os.getenv("ALLOWED_CHAT_IDS", "").split(",")))

async def security_check(update: Update) -> bool:
    chat_id = update.effective_chat.id
    if chat_id not in ALLOWED_CHAT_IDS:
        logger.warning(f"Unauthorized access attempt: {chat_id}")
        return False
    return True
```

### Данные в логах
В логах — только employee_id, никогда имена и тексты сообщений.
